# 兜底策略

本文档定义信息不足时的统一降级策略。

---

## 一、兜底原则

1. **不要卡住**：信息不足时优先产出通用草案，而非反复询问
2. **关键词优先**：根据关键词进行最小化推断
3. **通用降级可用**：允许进入通用工作流降级版
4. **后处理类不抢首轮**：后处理 skill 不得抢占首轮主路由

---

## 二、关键词兜底规则

当无法明确判断任务类型时，按以下关键词进行兜底路由：

| 关键词 | 兜底路由 | 说明 |
|-------|---------|------|
| 备课、教案 | lesson-planner | 教学设计类请求 |
| PPT、课件、幻灯片 | presentation | 演示文稿类请求 |
| 作业、练习、题、习题 | assignment | 练习生成类请求 |
| 讲解、错因、不要直接给答案 | tutor | 辅导讲解类请求 |
| 通知、评语、家长、家校沟通 | admin | 行政文案类请求 |
| 做一套教学内容、完整备课 | workflow-pack | 组合型请求 |

---

## 三、信息不足时的降级策略

### 学段信息缺失

| 缺失情况 | 降级策略 |
|---------|---------|
| 未提供学段 | 默认使用"初中"风格（通用性最强） |
| 未提供年级 | 默认使用"初二"（典型年级） |
| 未提供学科 | 默认使用"语文"（示例学科） |

**输出提示**：
> 💡 本内容采用通用风格生成。如需更精准适配，请补充"小学/初中/高中"信息。

### 任务参数缺失

| 任务类型 | 缺失字段 | 默认值 |
|---------|---------|-------|
| 教案 | 课时 | 40分钟 |
| PPT | 页数 | 8页 |
| 作业 | 题目数量 | 5题 |
| 作业 | 使用场景 | 课堂练习 |
| 行政 | 对象 | 家长 |
| 行政 | 语气 | 正式 |
| 辅导 | 学生水平 | 中等 |

### 组合请求信息不足

| 情况 | 降级策略 |
|-----|---------|
| "备课包"但未指定内容 | 使用默认备课包（教案+PPT+练习） |
| "家校沟通包"但未指定内容 | 使用默认沟通包（通知+反馈） |
| "辅导包"但未指定内容 | 使用默认辅导包（讲解+练习） |

---

## 四、通用工作流降级版

当请求模糊但可能需要组合输出时，可进入"通用工作流降级版"：

### 触发条件

| 条件 | 说明 |
|-----|------|
| "做一套教学内容" | 泛化组合请求 |
| "帮我准备上课内容" | 可能需要多模块 |
| "准备一节课的全部内容" | 隐含组合需求 |

### 降级版内容

| 工作流类型 | 降级版内容 |
|-----------|-----------|
| 备课包降级版 | 简版教案 + 精简PPT + 少量练习 |
| 家校沟通降级版 | 通用通知 + 简短反馈 |
| 辅导降级版 | 简要讲解 + 3道巩固题 |

### 降级版特点

1. **结构完整但精简**：保持结构但不冗长
2. **快速可用**：优先保证可用性
3. **提示可扩展**：告知用户可补充信息获得更详细版本

---

## 五、不能直接走复杂流程的情况

以下情况**不能**直接进入复杂工作流：

| 情况 | 处理方式 |
|-----|---------|
| 明确单一请求 | 路由到对应单技能，不强制组合 |
| 信息严重不足 | 先询问关键信息，不盲目降级 |
| 用户明确拒绝组合 | 尊重用户选择，使用单技能 |
| 后处理类请求 | 不走主路由，只做后置处理 |

---

## 六、后处理类不抢首轮入口说明

### 原则

后处理类 skill（quality-governor、observability）**不得**抢占首轮主路由。

### 触发时机

| Skill | 触发时机 | 说明 |
|-------|---------|------|
| quality-governor | 主任务完成后 | 对输出进行整形 |
| observability | 任务完成后 | 记录日志和统计 |

### 禁止行为

- ❌ 后处理类 skill 作为首轮路由目标
- ❌ 后处理类 skill 抢占业务类 skill
- ❌ 后处理类 skill 在主任务未完成时介入

### 正确流程

```
1. 首轮路由 → 业务类 skill（控制类/组合类/单技能类）
2. 执行主任务
3. 主任务完成后 → 检查是否需要后处理
4. 如需整形 → quality-governor
5. 如需记录 → observability
```

---

## 七、兜底流程图

```
步骤1：尝试匹配控制类
IF 匹配控制类触发词
   → 路由到控制类 skill
   → 结束

步骤2：尝试匹配组合类
IF 匹配组合类触发词 OR 包含多模块
   → 路由到 workflow-pack
   → 结束

步骤3：尝试匹配单技能类
IF 匹配某个单技能触发词
   → 路由到对应单技能
   → 结束

步骤4：使用关键词兜底
IF 包含特定关键词
   → 按关键词兜底规则路由
   → 结束

步骤5：使用通用降级
IF 请求模糊但可能需要组合
   → 进入通用工作流降级版
   → 结束

步骤6：询问用户
IF 完全无法推断
   → 询问用户具体需求
   → 结束
```

## 八、兜底示例

### 示例1：关键词兜底

**用户请求**："帮我做个课件"

**分析**：
- 包含"课件"关键词
- 无其他组合信息

**兜底路由**：`presentation`

---

### 示例2：通用降级

**用户请求**："帮我准备上课内容"

**分析**：
- 请求模糊
- "上课内容"隐含组合需求

**兜底路由**：`workflow-pack`（备课包降级版）

---

### 示例3：信息补全

**用户请求**："给我出几道题"

**分析**：
- 包含"题"关键词
- 缺失学科、年级、知识点

**兜底处理**：
1. 路由到 `assignment`
2. 自动补全默认值（语文、初中、5题）
3. 输出通用练习
4. 添加提示：可补充信息优化

---

### 示例4：完全无法推断

**用户请求**："帮我一下"

**分析**：
- 无任何关键词
- 无法推断任务类型

**兜底处理**：
1. 询问用户具体需求
2. 提供可选任务类型列表
