# Skill 边界定义

本文档定义每个 skill 的职责边界、强触发词、排除场景和易混淆对象。

---

## 一、控制类 Skill

### mode-switch

| 属性 | 定义 |
|-----|------|
| **职责** | 识别并执行"开启/关闭教师助手"口令 |
| **强触发词** | 开启教师助手、关闭教师助手、打开教师助手模式、退出教师助手模式 |
| **排除场景** | 业务任务请求（教案、PPT、作业等） |
| **易混淆对象** | 无 |
| **Use when** | 用户明确要求开启或关闭教师助手模式 |
| **Do not use when** | 用户请求的是具体业务任务 |
| **冲突让位规则** | 控制类优先，不与业务类竞争 |

### evaluation

| 属性 | 定义 |
|-----|------|
| **职责** | 运行评测/回归测试，验证核心能力是否退化 |
| **强触发词** | 运行评测、执行回归测试、运行测试用例 |
| **排除场景** | 业务任务请求 |
| **易混淆对象** | 作业/练习生成（assignment） |
| **Use when** | 用户明确要求运行评测或回归测试 |
| **Do not use when** | 用户请求的是生成作业或练习 |
| **冲突让位规则** | 控制类优先，不与业务类竞争 |

---

## 二、组合类 Skill

### workflow-pack

| 属性 | 定义 |
|-----|------|
| **职责** | 处理所有明确组合型请求，编排多模块串行执行 |
| **强触发词** | 备课包、完整备课、一整套课、教学整包、家校沟通包、辅导包、轻量课堂包 |
| **排除场景** | 明确单一产物请求（只要教案、只要PPT等） |
| **易混淆对象** | lesson-planner、presentation、assignment、admin、tutor |
| **Use when** | 用户请求包含多个模块组合（如"备课+PPT+练习"） |
| **Do not use when** | 用户明确只要单一产物 |
| **冲突让位规则** | 仅在单任务明确时让位给对应单技能 |

**组合模式识别**：

| 组合类型 | 触发条件 | 执行顺序 |
|---------|---------|---------|
| 备课包 | "备课包"/"完整备课"/"一整套课" 或 同时包含教案+PPT+练习 | lesson-planner → presentation → assignment |
| 家校沟通包 | "家校沟通包"/"通知+反馈" | admin(通知) → admin(反馈) |
| 辅导包 | "辅导包"/"讲解+练习" | tutor → assignment |
| 轻量课堂包 | "轻量课堂包"/"快速备课" | lesson-planner(简版) → presentation(简版) → assignment(少量) |

## 三、单技能类 Skill

### lesson-planner

| 属性 | 定义 |
|-----|------|
| **职责** | 生成教案、备课、课堂设计 |
| **强触发词** | 教案、备课、课堂设计、教学设计、备一节课 |
| **排除场景** | 完整备课包、PPT、作业、通知、辅导 |
| **易混淆对象** | workflow-pack（组合型备课请求）、presentation（PPT请求） |
| **Use when** | 用户明确只要教案或备课内容 |
| **Do not use when** | 用户请求的是完整备课包（含PPT/练习）或其他类型产物 |
| **冲突让位规则** | 遇到组合型请求让位给 workflow-pack |

**边界判断**：

| 用户请求 | 路由目标 |
|---------|---------|
| "给我备一节课" | lesson-planner |
| "帮我做个教案" | lesson-planner |
| "给我准备一整套课" | workflow-pack（备课包） |
| "备课 + PPT" | workflow-pack |
| "教案 + 练习" | workflow-pack |

### presentation

| 属性 | 定义 |
|-----|------|
| **职责** | 生成PPT、课件、幻灯片成品内容包 |
| **强触发词** | PPT、课件、幻灯片、讲课课件、讲课页 |
| **排除场景** | 完整备课包、单独教案、单独作业 |
| **易混淆对象** | workflow-pack（组合型请求含PPT）、lesson-planner（教案请求） |
| **Use when** | 用户明确只要PPT或课件内容 |
| **Do not use when** | 用户请求的是完整备课包或其他类型产物 |
| **冲突让位规则** | 遇到组合型请求让位给 workflow-pack |

**边界判断**：

| 用户请求 | 路由目标 |
|---------|---------|
| "给我做个PPT" | presentation |
| "帮我做个课件" | presentation |
| "备课 + PPT" | workflow-pack |
| "教案和PPT一起做" | workflow-pack |

### assignment

| 属性 | 定义 |
|-----|------|
| **职责** | 生成作业、练习、习题、分层题、巩固题 |
| **强触发词** | 作业、练习、习题、巩固题、分层题、出题、出几道 |
| **排除场景** | 讲解主任务、行政文案 |
| **易混淆对象** | tutor（讲解请求）、workflow-pack（讲解+练习组合） |
| **Use when** | 用户明确要求出题、生成练习或作业 |
| **Do not use when** | 用户请求的是讲解或辅导（不要直接给答案） |
| **冲突让位规则** | "讲解 + 练习"场景让位给组合流程 |

**边界判断**：

| 用户请求 | 路由目标 |
|---------|---------|
| "出5道练习题" | assignment |
| "布置一份作业" | assignment |
| "帮我讲这道题" | tutor |
| "讲解 + 出几道类似题" | workflow-pack（辅导包） |

### tutor

| 属性 | 定义 |
|-----|------|
| **职责** | 讲解、错因分析、启发式辅导 |
| **强触发词** | 讲题、讲解、错因、辅导、分析错误、怎么做、解题思路、不要直接给答案 |
| **排除场景** | 纯出题、纯通知、纯PPT |
| **易混淆对象** | assignment（出题请求）、workflow-pack（讲解+练习组合） |
| **Use when** | 用户要求讲解、辅导或分析错误 |
| **Do not use when** | 用户要求的是纯出题或生成练习 |
| **冲突让位规则** | "讲解 + 练习"组合请求让位给组合流程 |

**边界判断**：

| 用户请求 | 路由目标 |
|---------|---------|
| "帮我讲这道题" | tutor |
| "分析一下错因" | tutor |
| "出5道练习题" | assignment |
| "讲解 + 出类似题" | workflow-pack（辅导包） |

### admin

| 属性 | 定义 |
|-----|------|
| **职责** | 生成通知、评语、反馈、家校沟通草稿 |
| **强触发词** | 通知、评语、反馈、家校沟通、家长会、班级总结、写给家长、发群 |
| **排除场景** | 完整家校沟通包、教学内容 |
| **易混淆对象** | workflow-pack（多模块家校沟通请求） |
| **Use when** | 用户要求生成单独的通知、评语或反馈 |
| **Do not use when** | 用户请求的是完整家校沟通包或教学内容 |
| **冲突让位规则** | 多模块家校沟通请求让位给 workflow-pack |

**边界判断**：

| 用户请求 | 路由目标 |
|---------|---------|
| "写个通知" | admin |
| "帮我写评语" | admin |
| "通知 + 反馈一起做" | workflow-pack（家校沟通包） |
| "准备一套家校沟通内容" | workflow-pack |

## 四、后处理类 Skill

### quality-governor

| 属性 | 定义 |
|-----|------|
| **职责** | 后处理整形，确保输出符合统一规范 |
| **强触发词** | 无（自动触发） |
| **排除场景** | 首轮主路由 |
| **易混淆对象** | 无 |
| **Use when** | 主任务完成后需要整形 |
| **Do not use when** | 首轮路由阶段 |
| **冲突让位规则** | 不得抢首轮主路由，只做后置参与 |

### observability

| 属性 | 定义 |
|-----|------|
| **职责** | 后置记录，管理调试日志与使用统计 |
| **强触发词** | 无（自动触发） |
| **排除场景** | 首轮主路由 |
| **易混淆对象** | 无 |
| **Use when** | 任务完成后需要记录日志 |
| **Do not use when** | 首轮路由阶段 |
| **冲突让位规则** | 不得抢首轮主路由，只做后置参与 |

---

## 五、路由优先级总表

| 优先级 | 类型 | Skill | 说明 |
|-------|------|-------|------|
| 1 | 控制类 | mode-switch, evaluation | 优先拦截，不与业务类竞争 |
| 2 | 组合类 | workflow-pack | 次优先，处理组合型请求 |
| 3 | 单技能类 | lesson-planner, presentation, assignment, tutor, admin | 其后，处理单一任务 |
| 4 | 后处理类 | quality-governor, observability | 只后置参与，不抢首轮入口 |

---

## 六、边界判断流程

```
步骤1：检查是否为控制类请求
IF 匹配 mode-switch 或 evaluation 触发词
   → 路由到对应控制类 skill
   → 结束

步骤2：检查是否为组合类请求
IF 匹配 workflow-pack 触发词 OR 包含多模块组合
   → 路由到 workflow-pack
   → 结束

步骤3：检查是否为单技能请求
IF 匹配某个单技能触发词
   → 检查是否存在 skill 冲突
   IF 存在冲突
      → 按冲突矩阵裁决
   ELSE
      → 路由到对应单技能
   END
   → 结束

步骤4：信息不足，使用兜底策略
IF 无法明确判断
   → 按关键词兜底规则路由
   → 结束

步骤5：主任务完成后，检查是否需要后处理
IF 需要整形
   → 调用 quality-governor
IF 需要记录日志
   → 调用 observability
```
