# 错误日志模板

> 本模板定义错误日志的标准结构。

---

## 基本信息

| 字段 | 内容 |
|-----|------|
| 日期 | {YYYY-MM-DD} |
| 时间 | {HH:MM} |
| 错误ID | {可选，如 E001} |

---

## 错误分类

| 字段 | 内容 |
|-----|------|
| 任务类型 | {教案/PPT/作业/行政/辅导/工作流包} |
| 错误类别 | {结构不完整/风格不一致/降级触发/重写需求/流程中断} |
| 严重程度 | {低/中/高} |

---

## 错误详情

| 字段 | 内容 |
|-----|------|
| 触发原因 | {简要描述触发原因} |
| 影响范围 | {影响哪些模块或输出} |
| 用户可见 | {是/否} |

---

## 修复建议

| 字段 | 内容 |
|-----|------|
| 建议修复方向 | {简要描述修复方向} |
| 优先级 | {低/中/高} |
| 相关规则/模板 | {涉及的规则或模板文件} |

---

## 示例

```markdown
| 字段 | 内容 |
|-----|------|
| 日期 | 2026-03-01 |
| 时间 | 14:30 |
| 错误ID | E001 |
| 任务类型 | 教案 |
| 错误类别 | 结构不完整 |
| 严重程度 | 中 |
| 触发原因 | 教学目标字段缺失 |
| 影响范围 | 教案输出 |
| 用户可见 | 是 |
| 建议修复方向 | 检查 lesson-planner 输出验收规则 |
| 优先级 | 中 |
| 相关规则/模板 | skills/lesson-planner/SKILL.md |
```

---

## 记录原则

1. 只记录摘要，不记录完整敏感内容
2. 标明错误类别与触发原因
3. 提供简要修复方向
4. 便于后续检索与分析
