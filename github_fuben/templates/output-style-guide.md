# 统一输出风格指南

> 本文档定义教师助手系统所有模块的统一输出风格规范，确保多模块输出时风格一致、阅读顺畅。

---

## 一、标题层级规范

### 一级标题（H1）

- **用途**：任务结果名称，每个输出文件只有一个
- **格式**：`# {类型}：{主题}`
- **示例**：
  - `# 教案：《春》`
  - `# PPT课件：《春》`
  - `# 课堂练习：《春》`

### 二级标题（H2）

- **用途**：模块名称或主要部分
- **格式**：`## {部分名称}`
- **示例**：
  - `## 教学目标`
  - `## 教学过程`
  - `## 参考答案`

### 三级标题（H3）

- **用途**：子部分或具体环节
- **格式**：`### {环节名称}`
- **示例**：
  - `### 导入环节`
  - `### 新授环节`
  - `### 基础题`

### 四级标题（H4）

- **用途**：细节说明（谨慎使用，避免层级过深）
- **格式**：`#### {细节名称}`

## 三、默认语气说明

### 整体语气

- **专业**：用词准确，避免口语化随意表达
- **清晰**：结构分明，要点突出
- **可用**：老师可直接使用或稍作修改即可使用

### 不同学段语气差异

| 学段 | 语气特点 |
|-----|---------|
| 小学 | 活泼亲切、简单明了、多鼓励 |
| 初中 | 规范清晰、重点突出、适度严谨 |
| 高中 | 专业精炼、逻辑严密、深度分析 |

### 禁止的语气

- ❌ 过度口语化（如"咱们"、"吧"、"呢"过多）
- ❌ 过度书面化（如生僻词堆砌）
- ❌ 机器感（如"以下是生成的..."）
- ❌ 重复啰嗦（如反复"请注意"）

## 四、导读写法

### 何时需要导读

- 输出内容超过 100 行
- 来自多个模块的整包输出
- 工作流包结果

### 导读格式

```markdown
> **导读**：本{类型}包含{内容概要}，共{数量}部分。
```

### 导读示例

```markdown
> **导读**：本备课包包含教案、PPT课件、课堂练习三部分，已按初中风格生成。
```

```markdown
> **导读**：本辅导包包含错因分析、分步讲解、巩固练习三部分，适合中等基础学生。
```

## 五、结尾写法

### 默认结尾

使用简短、克制的结尾，不超过 2 行：

```markdown
---
*本内容由教师助手生成，可根据实际需要调整。*
```

### 禁止的结尾

- ❌ 每个模块都重复"如需我继续..."
- ❌ 过长的使用说明
- ❌ 多余的客套话

### 多模块整包结尾

整包输出只需一个统一结尾，不要每个子模块都加结尾：

```markdown
---
*本备课包由教师助手生成，包含教案、PPT、练习三部分，可根据实际需要调整使用。*
```

## 六、多模块拼接整合规则

### 拼接前检查

1. 各模块标题层级是否一致
2. 是否有重复内容需要去重
3. 语气是否统一
4. 过渡是否自然

### 整合步骤

1. **添加导读**：说明整包包含哪些内容
2. **统一标题**：确保一级标题准确描述整包
3. **模块分隔**：使用分隔线 `---` 分隔各模块
4. **去重处理**：删除各模块间重复的说明
5. **统一结尾**：整包只需一个结尾

## 七、列表与表格规范

### 列表规范

- 使用 `-` 作为无序列表符号
- 列表项首字对齐
- 嵌套列表缩进 2 空格

**正确示例**：
```markdown
- 知识目标：理解课文主要内容
- 能力目标：掌握修辞手法的运用
- 情感目标：感受作者对春天的热爱
```

### 表格规范

- 表头必须有
- 列对齐使用 `|` 分隔
- 简单内容用表格，复杂内容用列表

**正确示例**：
```markdown
| 环节 | 时间 | 内容 |
|-----|------|------|
| 导入 | 5分钟 | 情境导入 |
| 新授 | 25分钟 | 课文讲解 |
```

## 八、特殊标记规范

### 重点标记

- **加粗**：用于强调关键词
- `代码格式`：用于文件名、变量名等

### 提示框

使用引用格式表示提示：

```markdown
> 💡 提示：可根据班级实际情况调整题目难度。
```

### 警告框

```markdown
> ⚠️ 注意：请勿在课堂中直接展示答案。
```
