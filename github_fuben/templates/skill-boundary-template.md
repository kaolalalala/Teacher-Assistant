## 填写示例

### 示例：lesson-planner

| 属性 | 定义 |
|-----|------|
| **职责** | 生成教案、备课、课堂设计 |
| **强触发词** | 教案、备课、课堂设计、教学设计、备一节课 |
| **排除场景** | 完整备课包、PPT、作业、通知、辅导 |
| **易混淆对象** | workflow-pack（组合型备课请求）、presentation（PPT请求） |

#### Use when

- 用户明确只要教案或备课内容
- 用户请求单独的教学设计
- 用户说"给我备一节课"且无其他组合需求

#### Do not use when

- 用户请求的是完整备课包（含PPT/练习）
- 用户请求的是PPT或课件
- 用户请求的是作业或练习

#### 冲突让位规则

| 冲突对象 | 让位条件 | 让位方向 |
|---------|---------|---------|
| workflow-pack | 组合型请求（备课+PPT+练习） | 让位给 workflow-pack |
| presentation | 同时包含教案和PPT | 让位给 workflow-pack |

---

## 注意事项

1. **边界必须清晰**：每个 skill 的职责边界必须明确，避免模糊地带
2. **触发词要具体**：强触发词应该是明确的、不易混淆的关键词
3. **排除场景要完整**：列出所有不应触发的场景
4. **冲突规则要可执行**：让位规则必须具体、可操作
5. **示例要典型**：边界判断示例应覆盖常见场景
