# 模型回退策略文件（Model Fallback）

> 本文件定义模型不可用、成本过高或效果不稳时的回退规则。
> 多模型路由稳定化

---

## 一、回退触发场景

| 场景 | 触发条件 | 优先级 |
|------|----------|--------|
| 模型不可用 | API 超时、服务中断、配额耗尽 | P0 |
| 成本超限 | 单次/会话成本超过阈值 | P1 |
| 效果不稳定 | 输出质量波动、风格漂移 | P2 |
| 响应超时 | 生成时间超过阈值 | P2 |
| 用户主动切换 | 用户指定其他模型 | P3 |

---

## 二、模型回退链路

### 2.1 强模型回退链

```
strong（强模型）
    │
    ├── 不可用时 → 回退到 standard（标准模型）
    │
    └── standard 不可用 → 回退到 light（轻量模型）
                         │
                         └── light 不可用 → 返回错误并建议稍后重试
```

### 2.2 标准模型回退链

```
standard（标准模型）
    │
    ├── 不可用时 → 回退到 light（轻量模型）
    │              │
    │              └── 需评估质量影响
    │
    └── light 不可用 → 返回错误并建议稍后重试
```

### 2.3 轻量模型回退链

```
light（轻量模型）
    │
    └── 不可用时 → 返回错误并建议稍后重试
                   │
                   └── 可选：排队等待恢复
```

## 三、各 Skill 回退策略

### 3.1 workflow-pack

```
默认：strong
回退链：strong → standard → light → 错误

回退时处理：
- standard：保持整包输出，可能风格略有波动
- light：仅输出框架，提示用户可能需要人工润色
- 错误：记录日志，建议用户稍后重试

输出结构保持：
- 回退时保持输出结构不变
- 降级后提示用户质量可能下降
```

### 3.2 lesson-planner

```
默认：standard
回退链：standard → light → 错误

回退时处理：
- light：输出简版教案，提示用户可请求详细版本
- 错误：建议用户稍后重试

输出结构保持：
- 保持基本教案结构（目标、过程、评价）
- 降级后内容精简但结构完整
```

### 3.3 presentation

```
默认：standard
回退链：standard → light → 错误

回退时处理：
- light：输出简化版PPT框架，提示用户可请求详细版本
- 错误：建议用户稍后重试

输出结构保持：
- 保持PPT页码结构
- 降级后每页内容精简
```

### 3.4 assignment

```
默认：standard
回退链：standard → light → 错误

回退时处理：
- light：输出基础题型，提示用户可请求分层版本
- 错误：建议用户稍后重试

输出结构保持：
- 保持题目数量
- 降级后题目复杂度降低
```

### 3.5 tutor

```
默认：standard
回退链：standard → 错误（不允许降级到 light）

回退时处理：
- 错误：提示用户辅导功能暂时不可用，建议稍后重试

输出结构保持：
- 不允许降级保障辅导质量
```

### 3.6 admin

```
默认：light
回退链：light → 错误

回退时处理：
- 错误：建议用户稍后重试

输出结构保持：
- 保持通知/文案基本结构
```

### 3.7 quality-governor

```
默认：light
回退链：light → 错误

回退时处理：
- 错误：跳过整形步骤，返回原始输出

输出结构保持：
- 整形失败时返回原始内容
```

### 3.8 observability

```
默认：light
回退链：light → 错误

回退时处理：
- 错误：跳过日志记录，不影响主流程

输出结构保持：
- 记录失败不影响业务流程
```

### 3.9 evaluation

```
默认：standard
回退链：standard → light → 错误

回退时处理：
- light：输出简化评估报告，提示用户可请求详细版本
- 错误：建议用户稍后重试

输出结构保持：
- 保持评估维度结构
- 降级后分析深度降低
```

### 3.10 mode-switch

```
默认：light
回退链：light → 错误

回退时处理：
- 错误：提示用户模式切换暂时不可用

输出结构保持：
- 保持简洁响应格式
```

### 3.11 router-governor

```
默认：standard
回退链：standard → light → 错误

回退时处理：

- 错误：使用默认路由规则

输出结构保持：
- 保持路由判断输出格式
```

### 3.12 model-governor

```
默认：standard

回退时处理：

输出结构保持：
- 保持模型选择输出格式
```

### 3.13 head-teacher

```

回退链：standard → light → 错误

回退时处理：
```

## 四、回退时输出结构保持规则

### 4.1 结构保持原则

```
原则1：回退不改变输出格式
原则2：回退只影响内容深度/质量
原则3：回退后必须告知用户
原则4：关键结构字段不可缺失
```

### 4.2 结构保持检查清单

| 检查项 | 要求 | 示例 |
|--------|------|------|
| 输出格式 | 保持不变 | JSON 结构字段完整 |
| 必填字段 | 不可缺失 |教案的目标、过程字段 |
| 可选字段 | 可精简 | 扩展说明可省略 |
| 提示信息 | 必须添加 | "当前使用轻量模型，输出已精简" |

### 4.3 回退提示模板

```
【轻量模型回退提示】
当前使用轻量模型生成，输出内容已精简。如需详细版本，请稍后重试或联系管理员。

【标准模型回退提示】
当前使用标准模型生成，部分高级特性已简化。如需完整功能，请稍后重试。

【错误提示】
服务暂时不可用，请稍后重试。如问题持续，请联系管理员。
```

## 五、成本超限回退规则

### 5.1 成本阈值设置

| 场景 | 阈值 | 触发动作 |
|------|------|----------|
| 单次请求成本 | >0.1元 | 自动降级一档 |
| 会话累计成本 | >1元 | 强制使用轻量模型 |
| 用户日累计成本 | >10元 | 暂停服务并提示 |

### 5.2 成本回退流程

```
1. 检测成本超限
   │
   ├── 单次超限 → 降级一档后重试
   │
   ├── 会话超限 → 强制使用 light
   │
   └── 日累计超限 → 暂停并提示

2. 记录成本日志
   └── 写入成本监控表

3. 通知用户
   └── 返回降级提示
```

## 六、效果不稳定回退规则

### 6.1 不稳定检测指标

| 指标 | 阈值 | 检测方式 |
|------|------|----------|
| 输出长度波动 | >50% | 对比历史同类任务 |
| 风格漂移 | 明显 | 用户反馈/质量检测 |
| 格式错误率 | >10% | 结构化解析失败 |
| 重复率异常 | >30% | 文本相似度检测 |

### 6.2 不稳定回退流程

```
1. 检测到效果不稳定
   │
   ├── 记录不稳定事件
   │
   ├── 尝试回退到稳定档位
   │   └── 优先使用历史稳定档位
   │
   └── 通知用户
       └── 提示可能存在波动
```

## 七、响应超时回退规则

### 7.1 超时阈值

| 档位 | 超时阈值 | 回退动作 |
|------|----------|----------|
| strong | 60秒 | 降级到 standard 重试 |
| standard | 30秒 | 降级到 light 重试 |
| light | 15秒 | 返回错误 |

### 7.2 超时回退流程

```
1. 请求超时
   │
   ├── 记录超时事件
   │
   ├── 检查可回退档位
   │   ├── 有更低档位 → 降级重试
   │   └── 无更低档位 → 返回错误
   │
   └── 通知用户
       └── 提示响应较慢已切换模型
```

## 八、回退日志记录

### 8.1 日志字段

```json
{
  "timestamp": "2026-03-01T12:00:00Z",
  "session_id": "xxx",
  "skill": "lesson-planner",
  "original_tier": "standard",
  "fallback_tier": "light",
  "trigger_reason": "model_unavailable",
  "output_preserved": true,

}
```

### 8.2 日志分析用途

1. 统计各模型可用性
2. 分析回退触发频率
3. 优化回退策略
4. 成本节省统计
