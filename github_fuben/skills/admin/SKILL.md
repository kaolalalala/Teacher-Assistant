# SKILL.md - Admin（行政文案技能）

## 技能名称
admin

## 技能描述
专门负责生成行政文案的子技能，根据用户需求、学段特点、教师偏好生成通知、评语、活动文案、家校沟通等文本内容，输出为草稿状态。

## 触发条件
由 head-teacher 技能调用，当用户请求通知、评语、活动文案、家校沟通、家长会等内容时触发。

## 教师偏好读取

### 可参考偏好

| 偏好项 | 来源 | 影响内容 |
|-------|------|---------|
| 沟通语气 | profiles/teacher-profile.md | 正式/温和/简洁 |
| 文案长度偏好 | profiles/teacher-profile.md | 简短/标准/详细 |
| 是否偏好具体建议 | profiles/teacher-profile.md | 文案中建议的详细程度 |

### 可参考班级画像

| 画像项 | 来源 | 影响内容 |
|-------|------|---------|
| 家长关注度 | profiles/classroom-profile.md | 文案详细程度 |
| 沟通内容偏好 | profiles/classroom-profile.md | 问题反馈/进步表扬/全面汇报 |

### 偏好应用规则

1. **优先级**：当前任务明确要求 > 教师偏好 > 班级画像 > 通用默认规则
2. **语气适配**：
   - 偏好正式 → 规范用语，适合正式通知
   - 偏好温和 → 亲切表达，适合日常沟通
   - 偏好简洁 → 简明扼要，信息优先
3. **长度适配**：
   - 偏好简短 → 精简内容，突出要点
   - 偏好标准 → 按常规篇幅
   - 偏好详细 → 内容更丰富，细节更多
4. **班级适配**：
   - 家长关注度高 → 可适当增加细节和建议
   - 家长关注度一般 → 保持简洁，突出重点
5. **优雅降级**：偏好缺失时使用通用规则

### 应用示例

| 偏好组合 | 自动调整 |
|---------|---------|
| 温和语气 + 家长关注度高 | 亲切表达，内容详细，多给建议 |
| 简洁语气 + 家长关注度一般 | 简明扼要，突出核心信息 |
| 正式语气 + 高中学段 | 专业规范，目标导向 |

## 学段适配规则

### 输入参数

| 参数名 | 必需 | 说明 |
|-------|------|------|
| 学段 | 是 | 小学/初中/高中 |

### 小学场景风格

| 维度 | 特点 |
|-----|------|
| 语气 | 温和、亲切、鼓励 |
| 称呼 | "各位家长朋友"、"亲爱的家长" |
| 重点 | 习惯培养、兴趣激发、家校配合 |
| 表达 | 更口语化、生活化 |

**示例**：
> 亲爱的家长朋友们，本周孩子们在课堂上表现非常棒！...

**特点**：
- 更强调家校合作
- 更强调习惯培养
- 多用鼓励性表达
- 更贴近日常提醒

### 初中场景风格

| 维度 | 特点 |
|-----|------|
| 语气 | 平和、专业、诚恳 |
| 称呼 | "各位家长" |
| 重点 | 学习状态、习惯养成、阶段目标 |
| 表达 | 规范、清晰 |

**示例**：
> 各位家长：本周班级学习情况总结如下...

**特点**：
- 语气平衡
- 更强调学习状态和阶段目标
- 兼顾家长理解与配合

### 高中场景风格

| 维度 | 特点 |
|-----|------|
| 语气 | 简洁、专业、目标导向 |
| 称呼 | "各位家长" |
| 重点 | 学习效率、目标管理、考试准备 |
| 表达 | 精炼、重点突出 |

**示例**：
> 各位家长：现将本阶段学习情况反馈如下...

**特点**：
- 更强调效率、目标、考试导向
- 文案可更简洁直接
- 避免过度"儿童化"表达

### 学段适配检查

| 学段 | 检查要点 |
|-----|---------|
| 小学 | 是否温和亲切？是否有鼓励性？是否过于正式？ |
| 初中 | 是否专业诚恳？是否平衡？ |
| 高中 | 是否简洁目标导向？是否过于幼态？ |

```markdown
## 三、正文草稿
```

## 生成规则

### 内容要求

1. **语气得体**：根据对象和场景选择合适的语气
2. **信息完整**：时间、地点、事项等关键信息清晰
3. **格式规范**：符合通知/评语的常规格式
4. **避免敏感**：不使用绝对化、标签化表述
5. **草稿标记**：所有输出必须标注为草稿

### 语气风格对照

| 对象 | 推荐语气 | 示例特点 |
|-----|---------|---------|
| 家长 | 正式、温和 | 尊称开头，礼貌结尾 |
| 学生 | 亲切、鼓励 | 正面引导，具体肯定 |
| 教师 | 简洁、专业 | 条理清晰，重点突出 |
| 班级群 | 简洁、友好 | 信息明确，便于阅读 |

## 风险规则

### 必须遵守

1. **草稿标记**：所有输出必须标注"以下为草稿，请确认后再使用"
2. **不声称已发送**：不能表示"已发送""已通知"
3. **不自动发布**：不执行任何真实外发动作
4. **避免标签化**：学生评语避免使用"差生""问题学生"等标签

### 学生评语规范

- ✅ 使用具体、客观的描述
- ✅ 先肯定优点，再提出建议
- ✅ 使用鼓励性语言
- ❌ 避免绝对化表述（"总是""从不"）
- ❌ 避免负面标签
- ❌ 避免与他人比较

## 输出格式

- 格式：Markdown
- 语言：中文
- 风格：结构清晰、便于复制

## 边界限制

本技能：
- ✅ 生成行政文案草稿
- ✅ 提供多种语气版本
- ✅ 结构化输出
- ❌ 不自动发送消息
- ❌ 不声称已发送
- ❌ 不执行真实发布动作
- ❌ 不处理科研类任务

## 统一输出规范

### 标题格式统一

行政文案输出必须使用统一的标题格式：

```markdown
# 行政文案：{文案类型}
```

**示例**：
- `# 行政文案：家长会通知`
- `# 行政文案：学期总结`

### 小节命名固定

行政文案各部分顺序固定，命名稳定：

| 序号 | 固定命名 | 必需 |
|-----|---------|------|
| 1 | 文案类型 | ✅ |
| 2 | 适用对象 | ✅ |
| 3 | 正文草稿 | ✅ |
| 4 | 补充说明 | ❌ |
| 5 | 风格备注 | ✅ |

**禁止变体命名**：
- ❌ "文案信息" → 应为"文案类型"
- ❌ "目标对象" → 应为"适用对象"
- ❌ "正文内容" → 应为"正文草稿"

### 开头口吻统一

行政文案开头使用统一提示：

```markdown
> ⚠️ **以下为草稿，请确认后再使用。**
```

| 对象 | 开头规范 |

## 边界规则

### Use when

- 用户要求生成单独的通知、评语或反馈
- 用户说"写个通知"且无其他组合需求
- 用户说"帮我写评语"且未提及多模块

### Do not use when

- 用户请求的是完整家校沟通包
- 用户请求的是教学内容（教案/PPT/练习）
- 用户请求的是辅导讲解

### 冲突让位规则

| 冲突对象 | 让位条件 | 让位方向 |
|---------|---------|---------|
| workflow-pack | 多模块家校沟通请求 | 让位给 workflow-pack |

### 边界判断示例

| 用户请求 | 路由目标 | 判断依据 |
|---------|---------|---------|
| "写个通知" | admin | 单独通知 |
| "帮我写评语" | admin | 单独评语 |
| "通知 + 反馈一起做" | workflow-pack | 组合请求 |
| "家校沟通包" | workflow-pack | 组合请求 |

## 推荐模型档位

### 默认档位

| 属性 | 值 |
|-----|-----|
| 默认档位 | light（轻量模型） |
| 可升级 | standard |
| 可降级 | 不允许（已为最低档位） |
| 不推荐 | strong |

### 升级条件

满足以下条件时升级到标准模型：

| 触发场景 | 判断标准 |
|----------|----------|
| 正式通知公文 | 需要正式语气和规范格式 |
| 家校沟通套件 | 多份文案组合 |
| 重要活动通知 | 需要高度正式的表达 |
| 详细反馈报告 | 内容较长，需要结构化 |

### 不推荐强模型说明

**admin 技能不推荐使用强模型**，原因：

1. 行政文本复杂度有限：通知、评语等文本复杂度较低
2. 成本控制优先：行政文案无需高档模型
3. 质量已可保障：轻量/标准模型足以满足需求

### 档位选择示例

| 用户请求 | 推荐档位 | 选择原因 |
|---------|---------|---------|
| "写个通知" | light | 标准复杂度，使用默认档位 |
| "正式通知，要规范" | standard | 正式通知，升级标准模型 |
| "简单提醒一下" | light | 简单文案，保持轻量 |
| "家校沟通套件" | standard | 多份文案，升级标准模型 |
