# SKILL.md - Observability（可观测性与调试治理）

## 技能名称
observability

## 技能描述
管理系统的轻量调试日志与使用统计，帮助后续排查问题、优化规则、评估高频工作流和定位失败点。

## 触发条件

本技能不直接由用户触发，而是由主 agent 在以下情况下调用：

- 完成一个典型任务后
- 完成一个工作流包后
- 发生明显降级时
- 发生失败或输出不合格时
- 用户明确要求重写/调整时
- 需要生成每日/每周汇总时

## 核心职责

### 1. 告诉主 agent 哪些信息值得记录

| 信息类型 | 是否记录 | 说明 |
|---------|---------|------|
| 任务类型 | ✅ | 教案/PPT/作业/行政/辅导/工作流包 |
| 是否成功 | ✅ | 是/否 |
| 是否触发降级 | ✅ | 是/否 |
| 是否触发重写 | ✅ | 是/否 |
| 模式状态 | ✅ | ON/OFF |
| 上下文读取情况 | ✅ | 偏好/画像/档案 |
| 简要失败原因 | ✅ | 不超过20字 |
| 完整生成内容 | ❌ | 禁止记录 |
| 学生个人隐私 | ❌ | 禁止记录 |

### 2. 告诉主 agent 记录到哪个日志目录

| 记录类型 | 目录 | 命名格式 |
|---------|------|---------|
| 使用记录 | `logs/usage/` | `YYYY-MM.md` |
| 错误记录 | `logs/errors/` | `YYYY-MM-DD_{主题}.md` |
| 每日观察 | `logs/daily/` | `YYYY-MM-DD.md` |
| 每周总结 | `logs/weekly/` | `YYYY-WNN.md` |

### 3. 区分"使用统计"和"错误记录"

#### 使用统计（Usage）

用于记录：
- 哪类任务被调用
- 调用频率趋势
- 哪类工作流包最常用
- 模式开关使用情况

#### 错误记录（Error）

用于记录：
- 失败任务
- 降级触发
- 结构不完整
- 风格不一致
- 需要重生成的情况

## 记录分类详解

### 1. 使用记录（logs/usage/）

**记录时机**：
- 完成一个典型任务
- 完成一个工作流包

**记录格式**：
```markdown
| 字段 | 内容 |
|-----|------|
| 记录日期 | {YYYY-MM-DD} |
| 任务类型 | {教案/PPT/作业/行政/辅导/工作流包} |
| 学段 | {小学/初中/高中} |
| 是否成功 | {是/否} |
| 是否触发降级 | {是/否} |
| 模式状态 | {ON/OFF} |
| 是否读取偏好 | {是/否} |
| 是否属于工作流包 | {是/否} |
| 备注 | {简要备注} |
```

### 2. 错误记录（logs/errors/）

**记录时机**：
- 发生明显降级
- 发生失败或输出不合格
- 用户要求重写/调整

**记录格式**：
```markdown
| 字段 | 内容 |
|-----|------|
| 日期 | {YYYY-MM-DD} |
| 时间 | {HH:MM} |
| 任务类型 | {教案/PPT/作业/行政/辅导/工作流包} |
| 错误类别 | {结构不完整/风格不一致/降级触发/重写需求/流程中断} |
| 触发原因 | {简要描述} |
| 建议修复方向 | {简要建议} |
```

### 3. 每日观察（logs/daily/）

**记录时机**：每日结束时汇总

**记录格式**：
```markdown
# 每日观察：{YYYY-MM-DD}

## 今日概况

| 项目 | 数量 |
|-----|------|
| 总任务数 | {数量} |
| 成功任务 | {数量} |
| 降级触发 | {数量} |
| 重写需求 | {数量} |

## 高频任务

| 任务类型 | 次数 |
|---------|------|
| {类型} | {次数} |

## 常见问题

- {问题1}
- {问题2}

## 模式使用

| 模式 | 次数 |
|-----|------|
| ON | {次数} |
| OFF | {次数} |
```

### 4. 每周总结（logs/weekly/）

**记录时机**：每周结束时汇总

**记录格式**：
```markdown
# 每周总结：{YYYY-WNN}

## 本周概况

| 项目 | 数量 |
|-----|------|
| 总任务数 | {数量} |
| 成功率 | {百分比} |
| 降级率 | {百分比} |
| 重写率 | {百分比} |

## 高频任务

| 任务类型 | 次数 | 占比 |
|---------|------|------|
| {类型} | {次数} | {百分比} |

## 高频错误

| 错误类型 | 次数 | 说明 |
|---------|------|------|
| {类型} | {次数} | {说明} |

## 优化建议

- {建议1}
- {建议2}

## 需要调整的规则/模板

| 文件 | 问题 | 建议 |
|-----|------|------|
| {文件路径} | {问题} | {建议} |
```

## 记录原则

### 1. 记录"摘要"，不记录"完整原文"

**可以记录**：
- "初中备课包，成功，模式ON，使用偏好，未降级"

**禁止记录**：
- 完整教案正文
- 完整学生题目原文
- 完整家校沟通全文

### 2. 不记录敏感信息

**禁止记录**：
- 学生个人隐私
- 明显可识别个人信息
- 高敏感个体评价

### 3. 优先记录"可优化信息"

最值得记录的是：
- 经常失败的地方
- 经常重写的地方
- 经常触发降级的地方
- 高频任务与高频工作流

### 4. 日志不能干扰主任务

即使启用观测记录，也不能让日志写入本身破坏主流程的可用性。

## 错误类别定义

| 错误类别 | 说明 | 示例 |
|---------|------|------|
| 结构不完整 | 输出缺少必要字段 | 教案缺少教学目标 |
| 风格不一致 | 输出与学段/偏好不匹配 | 高中教案过于幼态 |
| 降级触发 | 信息不足导致使用默认值 | 年级未指定，使用初中默认 |
| 重写需求 | 用户要求重写/调整 | 用户要求"更正式" |
| 流程中断 | 工作流包执行中断 | 备课包只完成了教案 |

## 与其他技能的关系

| 技能 | 关系 |
|-----|------|
| head-teacher | 主 agent 调用本技能记录日志 |
| 所有业务技能 | 业务技能执行结果由本技能记录 |

## 边界限制

### 支持

- ✅ 记录轻量使用摘要
- ✅ 记录轻量错误摘要
- ✅ 区分 usage/error/daily/weekly 日志
- ✅ 生成每日/每周汇总

### 不支持

- ❌ 复杂数据库
- ❌ 外部监控系统接入
- ❌ 完整聊天逐字存档
- ❌ 敏感个人隐私记录
- ❌ 实时统计分析

## 示例

### 示例1：记录使用摘要

```
主 agent 完成一个备课包任务后：

1. 调用 observability 技能
2. 记录到 logs/usage/2026-03.md
3. 内容：
   - 任务类型：备课包
   - 学段：初中
   - 是否成功：是
   - 模式状态：ON
   - 备注：初二语文《春》
```

### 示例2：记录错误摘要

```
主 agent 发现教案结构不完整后：

1. 调用 observability 技能
2. 记录到 logs/errors/2026-03-01_教案结构问题.md
3. 内容：
   - 错误类别：结构不完整
   - 触发原因：教学目标字段缺失
   - 建议修复：检查 lesson-planner 验收规则
```

### 示例3：生成每日汇总

```
每日结束时：

1. 读取 logs/usage/ 当日记录
2. 读取 logs/errors/ 当日记录
3. 汇总生成 logs/daily/2026-03-01.md
4. 内容包括：
   - 今日概况统计
   - 高频任务排行
   - 常见问题列表
   - 模式使用情况
   ```

## 边界规则

### Use when

- 任务完成后需要记录日志
- 需要生成每日/每周汇总
- 发生错误或降级需要记录

### Do not use when

- 首轮路由阶段
- 作为主路由目标
- 抢占业务类 skill

### 冲突让位规则

本技能为**后处理类**，不得抢占首轮主路由：

| 场景 | 行为 |
|-----|------|
| 首轮路由 | 不参与，让位给业务类 skill |
| 任务完成后 | 可参与，记录日志和统计 |

### 后处理约束

- ❌ 不得作为首轮路由目标
- ❌ 不得抢占业务类 skill
- ❌ 不得在主任务未完成时介入
- ✅ 只在任务完成后作为后置步骤参与

## 推荐模型档位

### 默认档位

| 属性 | 值 |
|-----|-----|
| 默认档位 | light（轻量模型） |
| 可升级 | 不允许 |
| 可降级 | 不允许（已为最低档位） |
| 不推荐 | standard, strong |

### 默认轻量模型原因

**observability 默认使用轻量模型**，原因：

1. 日志记录简单：摘要记录、统计汇总等任务复杂度极低
2. 成本控制优先：日志记录不应占用任何高档资源
3. 非核心业务：日志是辅助功能，不应影响主任务成本

### 禁止升级说明

**observability 不允许升级模型**，原因：

1. 任务极其简单：日志记录、统计汇总无需强推理
2. 成本敏感：日志功能应保持最低成本
3. 不影响主流程：日志质量要求不高

### 档位选择示例

| 场景 | 推荐档位 | 选择原因 |
|---------|---------|---------|
| 记录使用摘要 | light | 所有场景使用轻量模型 |
| 记录错误日志 | light | 所有场景使用轻量模型 |
| 生成每日汇总 | light | 所有场景使用轻量模型 |
| 生成每周总结 | light | 所有场景使用轻量模型 |
