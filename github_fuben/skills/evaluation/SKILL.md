# SKILL.md - Evaluation（评测治理技能）

## 技能名称

evaluation

## 技能描述

评测治理层技能，负责运行固定测试用例、检查预期行为、记录测试结果、判断是否出现回归退化。该技能不替代业务技能，而是为系统提供质量保障能力。

## 触发条件

- 用户明确要求"运行回归测试"、"执行评测"
- 规则更新后需要验证
- 阶段验收时需要回归检查

## 核心职责

1. **运行固定测试题**：按 eval/cases/ 中的用例执行
2. **检查预期行为**：对照 eval/baselines/ 检查行为正确性
3. **记录测试结果**：将结果写入 eval/results/
4. **判断回归退化**：识别是否破坏已有能力

## 评测原则

### 不做逐字比对

评测标准不是"和某段参考文本完全一致"，而是检查：

- 路由是否正确
- 结构是否完整
- 顺序是否正确
- 模式是否生效
- 是否明显退化

### 重点检查行为与结构

| 检查维度 | 说明 |
|---------|------|
| 路由正确性 | 任务是否路由到正确的模块 |
| 输出结构 | 必需字段是否完整 |
| 执行顺序 | 工作流包是否按正确顺序执行 |
| 模式行为 | ON/OFF 状态下行为是否正确 |
| 降级情况 | 是否触发异常降级 |

### 允许轻微措辞变化

只要行为正确、结构合理，即可视为通过或部分通过。

### 评测结果必须可追溯

每轮评测都留下结果文件，而不是只口头判断。

## 判定结果分类

### Pass（通过）

满足以下全部条件：

- 核心行为符合预期
- 输出结构完整
- 无明显退化
- 路由正确
- 模式行为正确

### Partial Pass（部分通过）

满足以下条件：

- 能完成任务
- 但结构、风格或顺序有轻微偏差
- 不影响主能力可用性

典型情况：

- 输出结构完整但风格略有偏差
- 工作流包执行顺序正确但过渡语不够自然
- 模式行为正确但读取了不必要的文件

### Fail（不通过）

出现以下任一情况：

- 路由错误（如教案任务被路由到 PPT 模块）
- 结构缺失严重（缺少必需字段）
- 模式开关失效（ON/OFF 行为不正确）
- 工作流顺序错误（未按定义顺序执行）
- 核心功能不可用（无法完成任务）

## 评测执行流程

```
步骤1：准备评测环境
   → 确认系统状态
   → 确认模式状态

步骤2：读取测试用例
   → 从 eval/cases/ 读取固定测试用例
   → 按用例顺序执行

步骤3：执行单个测试用例
   → 按用例输入指令执行任务
   → 记录实际输出
   → 对照 baseline 检查

步骤4：判定结果
   → Pass / Partial Pass / Fail
   → 记录偏差点

步骤5：记录评测结果
   → 写入 eval/results/
   → 包含问题摘要和建议修复项

步骤6：汇总报告
   → 统计通过率
   → 列出问题清单
   → 提供修复建议
```

## 评测报告格式

评测结果应包含以下内容：

```markdown
# 评测结果报告

## 评测信息
- 评测日期：YYYY-MM-DD
- 评测范围：[用例范围]
- 触发原因：[原因]

## 评测概览
- 总用例数：X
- 通过数：X
- 部分通过数：X
- 不通过数：X
- 通过率：XX%

## 各用例结果
[详细结果列表]

## 问题摘要
[发现的问题汇总]

## 建议修复项
[修复建议]

## 结论
[总体判定]
```

## 使用场景

当出现以下情况时使用本技能：

| 场景 | 说明 |
|-----|------|
| 更新 AGENTS.md | 验证主控规则是否破坏已有能力 |
| 更新某个 SKILL.md | 验证对应模块是否正常工作 |
| 调整模式开关规则 | 验证模式行为是否正确 |
| 调整输出结构 | 验证输出格式是否正确 |
| 新增重要模板 | 验证模板应用是否正确 |
| 阶段验收 | 验证本阶段能力是否完整 |

## 注意事项

1. **不替代业务技能**：本技能用于质量保障，不直接生成业务输出
2. **保持轻量**：评测过程不应过于复杂，影响系统可用性
3. **结果可追溯**：每轮评测都应留下结果文件
4. **关注退化**：重点检查是否破坏已有能力
5. **持续完善**：测试用例和 baseline 应随系统演进持续更新

## 边界规则

### Use when

- 用户明确要求运行评测或回归测试
- 用户说"运行评测"、"执行回归测试"
- 规则更新后需要验证能力是否退化

### Do not use when

- 用户请求的是业务任务（教案/PPT/作业等）
- 用户请求的是生成练习或作业

### 冲突让位规则

本技能为**控制类**，优先级最高：

| 场景 | 行为 |
|-----|------|
| 控制类请求 | 优先拦截，不与业务类竞争 |
| 业务类请求 | 让位给对应业务类 skill |

### 控制类优先规则

- ✅ 控制类一律优先于业务类
- ✅ 不与业务类 skill 竞争
- ✅ 只响应评测/回归测试口令
