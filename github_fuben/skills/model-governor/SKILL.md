# SKILL.md - Model Governor（模型治理技能）

## 技能名称
model-governor

## 技能描述
模型治理层技能，负责根据 skill 类型为任务分配默认模型档位、处理模型升级/降级、避免同类任务频繁换模、定义模型不可用时的回退顺序，保障输出质量、成本和风格之间的平衡。

## 触发条件
- 由 head-teacher 在完成 skill 路由后调用
- 同一 skill 可以接多个不同模型时
- 不同任务复杂度差异明显时
- 需要做多模型实验但又不希望系统失控时
- 出现"这次质量好、下次变差"的模型漂移问题时

## 模型治理能力

### 核心职责

1. **默认模型分配**：根据 skill 类型为任务分配默认模型档位
2. **升级/降级判断**：根据任务复杂度判断是否需要升级或降级
3. **稳定性保障**：避免同类任务频繁换模
4. **回退策略执行**：定义模型不可用时的回退顺序
5. **成本与质量平衡**：保障输出质量、成本和风格之间的平衡

### 模型档位定义

| 档位 | 代号 | 适用场景 | 特点 |
|------|------|----------|------|
| 强模型 | `strong` | 复杂编排、组合任务、长输出整合、高一致性要求 | 高质量、高成本、强推理 |
| 标准模型 | `standard` | 大多数稳定文本生成、单技能常规输出 | 平衡质量与成本 |
| 轻量模型 | `light` | 简单改写、简短文案、格式整理、成本敏感场景 | 低成本、快速响应 |

### 模型选择流程

```
步骤1：接收 skill 路由结果
   → 获取目标 skill 名称
   → 获取任务描述

步骤2：查询默认模型档位
   → 读取 models/model-map.md
   → 获取该 skill 的默认模型档位

步骤3：检查历史模型使用
   → 查询同类任务历史模型使用记录
   IF 有历史记录
      → 优先使用历史档位（稳定性优先）
   ELSE
      → 使用默认档位

步骤4：评估任务复杂度
   → 分析任务描述
   → 判断复杂度等级（高/中/低）

步骤5：判断是否需要升级
   IF 复杂度高 AND 满足升级条件
      → 计算升级收益
      IF 收益明显
         → 升级到更高档位
      ELSE
         → 保持当前档位
   ELSE
      → 保持当前档位

步骤6：判断是否需要降级
   IF 复杂度低 AND 允许降级 AND 满足降级条件
      → 评估降级风险
      IF 风险可接受
         → 降级到更低档位
      ELSE
         → 保持当前档位
   ELSE
      → 保持当前档位

步骤7：检查稳定性约束
   → 检查同类任务档位波动频率
   IF 波动频繁
      → 锁定默认档位
   → 检查升降级来回切换
   IF 切换频繁
      → 冻结档位选择

步骤8：检查成本预算
   → 检查当前会话成本
   IF 成本超限
      → 触发降级

步骤9：输出模型选择结果
   → 返回选择的模型档位
   → 返回选择原因
```

### 升级判断规则

#### 升级触发条件

| 触发场景 | 判断标准 | 适用 Skill |
|----------|----------|------------|
| 组合型工作流 | 多 Skill 编排、整包输出 | workflow-pack |
| 长文本整合 | 输出预计 >2000字 | lesson-planner, presentation |
| 结构复杂 | 多层级、多模块结构 | lesson-planner, presentation, assignment |
| 强一致性要求 | 风格/格式需高度统一 | workflow-pack, presentation |
| 复杂推理 | 多步骤逻辑链 | tutor, evaluation, router-governor |
| 启发式讲解 | 需要深度引导设计 | tutor |
| 复杂评估 | 多维度综合判断 | evaluation |
| 路由冲突严重 | 多 Skill 边界模糊 | router-governor, model-governor |

#### 升级判断流程

```
1. 检查任务复杂度
   ├── 复杂度 >= 高 → 进入升级评估
   └── 复杂度 < 高 → 保持默认

2. 评估升级收益
   ├── 质量提升明显 → 允许升级
   └── 质量提升有限 → 保持默认

3. 检查成本预算
   ├── 预算充足 → 执行升级
   └── 预算紧张 → 记录并保持默认
```

### 降级判断规则

#### 降级触发条件

| 触发场景 | 判断标准 | 适用 Skill |
|----------|----------|------------|
| 任务简单 | 复杂度 < 中 | lesson-planner, presentation, assignment |
| 输出短 | 预计输出 <500字 | lesson-planner, presentation |
| 格式整理 | 仅需结构调整 | quality-governor |
| 简单改写 | 文本润色、格式化 | admin |
| 状态切换 | 模式口令处理 | mode-switch |
| 日志记录 | 摘要、统计记录 | observability |
| 简单统计 | 数值汇总 | evaluation |

#### 降级判断流程

```
1. 检查任务复杂度
   ├── 复杂度 <= 低 → 进入降级评估
   └── 复杂度 > 低 → 保持默认

2. 评估降级风险
   ├── 质量影响可接受 → 允许降级
   └── 质量影响明显 → 保持默认

3. 检查用户期望
   ├── 用户无特殊要求 → 执行降级
   └── 用户期望高质量 → 保持默认
```

#### 禁止降级场景

以下场景**禁止降级**：

1. **tutor（答疑辅导）**：辅导质量优先，不允许降级到 light
2. **model-governor（模型治理）**：判断稳定性优先，不允许降级
3. **已为最低档位**：light 档位不允许继续降级
4. **用户明确要求高质量**：尊重用户期望
5. **历史同类任务使用高档模型**：保持一致性

### 稳定性保障规则

#### 稳定性原则

```
原则1：同类任务使用相同模型档位
原则2：同一会话内保持模型一致性
原则3：同一用户跨会话保持风格统一
原则4：模型切换需有明确理由并记录
```

#### 频繁换模检测

| 检测项 | 阈值 | 处理方式 |
|--------|------|----------|
| 同类任务档位波动 | 3次/会话 | 锁定默认档位 |
| 同一Skill换模频率 | >50%/会话 | 强制使用默认档位 |
| 升降级来回切换 | 2次/任务流 | 冻结档位选择 |

### 回退策略

#### 回退触发场景

| 场景 | 触发条件 | 优先级 |
|------|----------|--------|
| 模型不可用 | API 超时、服务中断、配额耗尽 | P0 |
| 成本超限 | 单次/会话成本超过阈值 | P1 |
| 效果不稳定 | 输出质量波动、风格漂移 | P2 |
| 响应超时 | 生成时间超过阈值 | P2 |

#### 回退链路

```
strong（强模型）
    │
    ├── 不可用时 → 回退到 standard（标准模型）
    │
    └── standard 不可用 → 回退到 light（轻量模型）
                         │
                         └── light 不可用 → 返回错误并建议稍后重试
```

#### 回退时处理

- 保持输出结构不变
- 告知用户当前使用的模型档位
- 记录回退事件到日志

### 成本控制策略

#### 成本预算分配

| 档位 | 单次成本系数 | 建议使用比例 |
|------|--------------|--------------|
| strong | 3.0 | ≤20% |
| standard | 1.5 | 50-60% |
| light | 1.0 | ≥20% |

#### 成本控制规则

1. **强模型配额限制**：每会话强模型使用不超过 20%
2. **批量任务降级**：批量生成任务优先使用轻量模型
3. **成本预警**：单会话成本超阈值时自动降级
4. **用户配额**：支持按用户设置成本上限

### 模型选择决策树

```
开始模型选择
│
├── 1. 确定 Skill 类型
│   └── 查询 model-map.md 获取默认档位
│
├── 2. 检查历史使用
│   ├── 有同类任务历史 → 优先使用历史档位
│   └── 无历史 → 使用默认档位
│
├── 3. 评估任务复杂度
│   ├── 复杂度高 → 检查升级条件
│   ├── 复杂度中 → 保持当前档位
│   └── 复杂度低 → 检查降级条件
│
├── 4. 检查稳定性约束
│   ├── 同类任务档位波动 → 锁定默认档位
│   └── 档位稳定 → 继续判断
│
├── 5. 检查成本预算
│   ├── 预算充足 → 执行选择
│   └── 预算紧张 → 评估降级可能
│
└── 6. 输出模型选择结果
    └── 记录选择原因
```

### 策略执行优先级

| 优先级 | 规则类型 | 说明 |
|--------|----------|------|
| P0 | 禁止规则 | 不推荐档位绝对禁止 |
| P1 | 稳定性规则 | 同类任务保持一致 |
| P2 | 升级规则 | 满足条件才升级 |
| P3 | 降级规则 | 评估影响后降级 |
| P4 | 成本规则 | 预算约束下调整 |

## 边界限制

### 支持
- ✅ 默认模型档位分配
- ✅ 升级/降级判断
- ✅ 稳定性保障
- ✅ 回退策略执行
- ✅ 成本控制

### 不支持
- ❌ 完全自动自由选模
- ❌ 复杂动态学习型模型调度
- ❌ 强制绑定真实 API 变更逻辑
- ❌ 跨会话模型偏好学习
