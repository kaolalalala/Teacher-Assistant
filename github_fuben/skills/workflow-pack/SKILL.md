# SKILL.md - Workflow Pack（工作流包技能）

## 技能名称
workflow-pack

## 技能描述
工作流包技能是"组合调度层"，负责识别高层次教学场景，将多个基础模块封装成可复用的高频套餐，实现"一次请求 → 一整套教学产物"的输出能力。

## 触发条件
由 head-teacher 技能调用，当用户请求属于"组合型高层任务"时触发。

## 核心职责

1. **识别工作流类型**：判断用户请求属于哪类工作流包
2. **编排执行顺序**：确定子模块的执行顺序
3. **协调风格统一**：确保各子模块输出风格一致
4. **整合最终输出**：按规范顺序整合展示

---

## 工作流包类型

### 1. 备课包（Lesson Pack）

#### 触发词

```
备课包、完整备课、一整套课、备课+PPT+练习、教学整包、课堂整包
```

#### 默认包含

| 序号 | 模块 | 技能 | 说明 |
|-----|------|------|------|
| 1 | 教案 | lesson-planner | 结构完整的教学设计 |
| 2 | PPT/课件 | presentation | 逐页成品内容包 |
| 3 | 练习 | assignment | 默认课堂练习（4-6题） |

#### 执行流程

```
步骤1：调用 lesson-planner
   ↓ 传入：学科、年级、课题、学段、偏好
   ↓ 输出：完整教案
   ↓ 提取：教案摘要

步骤2：调用 presentation
   ↓ 传入：教案摘要、学段、偏好
   ↓ 输出：PPT成品内容包

步骤3：调用 assignment
   ↓ 传入：教案摘要、学段、偏好
   ↓ 输出：配套练习

步骤4：整合输出
   ↓ 按顺序：教案 → PPT → 练习
   ↓ 添加使用说明
```

#### 裁剪规则

| 用户要求 | 执行调整 |
|---------|---------|
| 不要PPT | 跳过 presentation |
| 不要练习 | 跳过 assignment |
| 只要教案 | 只执行 lesson-planner |
| 课后作业 | assignment 改为课后作业模式 |

---

### 2. 家校沟通包（Parent Communication Pack）

#### 触发词

```
家校沟通包、一套家长沟通、通知+反馈、家长会内容包、家长沟通整包
```

#### 默认包含

| 序号 | 模块 | 技能 | 说明 |
|-----|------|------|------|
| 1 | 通知草稿 | admin | 班级通知或活动通知 |
| 2 | 学习反馈草稿 | admin | 学生学习情况反馈 |
| 3 | 家长沟通建议 | admin | 可选，建议沟通话术 |

#### 执行流程

```
步骤1：调用 admin（通知）
   ↓ 传入：文案类型=通知、对象=家长、场景、偏好
   ↓ 输出：通知草稿

步骤2：调用 admin（反馈）
   ↓ 传入：文案类型=反馈、对象=家长、场景、偏好
   ↓ 输出：学习反馈草稿

步骤3：生成沟通建议
   ↓ 基于通知和反馈内容
   ↓ 输出：建议沟通话术

步骤4：整合输出
   ↓ 按顺序：通知 → 反馈 → 建议
   ↓ 标注草稿状态
```

#### 裁剪规则

| 用户要求 | 执行调整 |
|---------|---------|
| 只要通知 | 只生成通知草稿 |
| 不要建议 | 跳过沟通建议 |
| 正式语气 | 全部使用正式语气 |

---

### 3. 辅导包（Tutoring Pack）

#### 触发词

```
辅导包、一套辅导内容、讲解+练习、学生辅导整包、辅导整包
```

#### 默认包含

| 序号 | 模块 | 技能 | 说明 |
|-----|------|------|------|
| 1 | 错因分析 | tutor | 分析常见错误原因 |
| 2 | 分步讲解 | tutor | 启发式讲解思路 |
| 3 | 巩固练习 | assignment | 相似题/巩固题（3-5题） |
| 4 | 教师建议 | tutor | 给教师的辅导建议 |

#### 执行流程

```
步骤1：调用 tutor
   ↓ 传入：题目、学科、学生水平、偏好
   ↓ 输出：错因分析 + 分步讲解 + 教师建议
   ↓ 提取：知识点、讲解要点

步骤2：调用 assignment
   ↓ 传入：知识点、难度、数量
   ↓ 输出：巩固练习

步骤3：整合输出
   ↓ 按顺序：错因分析 → 讲解 → 练习 → 建议
   ↓ 添加使用说明
```

#### 裁剪规则

| 用户要求 | 执行调整 |
|---------|---------|
| 不要练习 | 跳过 assignment |
| 直接给答案 | 在讲解末尾提供答案 |
| 只要练习 | 只执行 assignment |

---

### 4. 轻量课堂包（Light Classroom Pack）

#### 触发词

```
轻量课堂包、快速备课、简版课堂、轻量备课
```

#### 默认包含

| 序号 | 模块 | 技能 | 说明 |
|-----|------|------|------|
| 1 | 简版教案 | lesson-planner | 聚焦核心的精简教案 |
| 2 | 课堂PPT | presentation | 精简版PPT内容 |
| 3 | 课堂练习 | assignment | 少量练习（3-4题） |

#### 执行流程

```
步骤1：调用 lesson-planner（简版模式）
   ↓ 传入：学科、年级、课题、模式=简洁
   ↓ 输出：简版教案

步骤2：调用 presentation（简版模式）
   ↓ 传入：教案摘要、模式=简洁
   ↓ 输出：精简PPT

步骤3：调用 assignment（少量模式）
   ↓ 传入：知识点、数量=3-4
   ↓ 输出：少量练习

步骤4：整合输出
```

## 识别规则

### 工作流包识别优先级

```
优先级1：明确的工作流包关键词
IF 输入包含 ("备课包" OR "完整备课" OR "一整套课")
   → 备课包

ELSE IF 输入包含 ("家校沟通包" OR "一套家长沟通" OR "通知+反馈")
   → 家校沟通包

ELSE IF 输入包含 ("辅导包" OR "一套辅导内容" OR "讲解+练习")
   → 辅导包

ELSE IF 输入包含 ("轻量课堂包" OR "快速备课" OR "简版课堂")
   → 轻量课堂包

优先级2：组合型请求推断
ELSE IF 输入包含 ("备课" OR "教案") AND 包含 ("PPT" OR "课件") AND 包含 ("练习" OR "作业")
   → 备课包

ELSE IF 输入包含 ("通知") AND 包含 ("反馈")
   → 家校沟通包

ELSE IF 输入包含 ("讲解" OR "辅导") AND 包含 ("练习" OR "巩固")
   → 辅导包

优先级3：基础任务（非工作流包）
ELSE
   → 按原有规则识别基础任务
```

## 执行规则

### 串行执行

所有工作流包必须串行执行，不允许并行：

```
模块1 → 模块2 → 模块3 → 整合输出
```

### 上下文传递

前一模块的输出摘要需传递给后续模块：

| 工作流包 | 传递内容 |
|---------|---------|
| 备课包 | 教案摘要 → PPT、练习 |
| 辅导包 | 知识点、讲解要点 → 练习 |

### 风格统一

- 所有子模块使用相同的学段设置
- 所有子模块使用相同的教师偏好
- 语言风格保持一致

---

## 降级规则

### 信息不足时的处理

| 工作流包 | 降级策略 |
|---------|---------|
| 备课包 | 生成通用版教案 + 通用版PPT + 基础练习 |
| 家校沟通包 | 生成通用版通知草稿 + 通用版反馈草稿 |
| 辅导包 | 生成通用讲解框架 + 基础巩固题 |
| 轻量课堂包 | 生成最简版教案 + 精简PPT + 少量练习 |

### 子模块失败时的处理

1. 尝试重新执行（最多1次）
2. 若仍失败，跳过该模块，继续其他模块
3. 在最终输出中标注缺失部分

---

## 验收规则

### 完整性检查

| 工作流包 | 必需检查项 |
|---------|-----------|
| 备课包 | 教案完整、PPT页数合理、练习数量符合要求 |
| 家校沟通包 | 通知结构完整、反馈内容合理 |
| 辅导包 | 讲解步骤清晰、练习与知识点相关 |

### 一致性检查

- 学段风格是否统一
- 语言风格是否统一
- 是否存在重复内容
- 是否存在冲突信息

## 输出格式

### 整合输出规范

```markdown
# {工作流包名称}：{主题}

---
生成时间：{时间}
[元信息]
---

## 一、{第一个模块名称}

[内容]

---

## 二、{第二个模块名称}

[内容]

---

[更多模块...]

---
💡 **使用说明**：[使用建议]
```

---

## 边界限制

本技能：
- ✅ 识别工作流包类型
- ✅ 编排子模块执行顺序
- ✅ 协调风格统一
- ✅ 整合最终输出
- ❌ 不替代基础技能的内部逻辑
- ❌ 不新增独立输出目录
- ❌ 不执行并行调度
- ❌ 不自动发送或发布

## 示例

### 示例1：备课包

**输入**：
```
给初二语文《春》准备一个完整备课包
```

**处理**：
1. 识别为备课包
2. 调用 lesson-planner → 教案
3. 调用 presentation → PPT
4. 调用 assignment → 练习
5. 整合输出

### 示例2：家校沟通包

**输入**：
```
帮我准备一套期中考试后的家校沟通内容
```

**处理**：
1. 识别为家校沟通包
2. 调用 admin → 通知草稿
3. 调用 admin → 反馈草稿
4. 生成沟通建议
5. 整合输出

### 示例3：辅导包

**输入**：
```
给这道题准备一套辅导内容：解方程 x²-5x+6=0
```

**处理**：
1. 识别为辅导包
2. 调用 tutor → 错因分析 + 讲解
3. 调用 assignment → 巩固练习
4. 整合输出

### 示例4：裁剪请求

**输入**：
```
备课包，但不要PPT
```

**处理**：
1. 识别为备课包
2. 检测到裁剪要求"不要PPT"
3. 只执行 lesson-planner + assignment
4. 整合输出（教案 + 练习）

## 统一输出规范

### 整包输出必须带导读

工作流包输出必须在开头增加简短导读：

```markdown
> **导读**：本{包类型}包含{内容概要}，共{数量}部分，已按{学段}风格生成。
```

**示例**：
```markdown
> **导读**：本备课包包含教案、PPT课件、课堂练习三部分，已按初中风格生成。
```

### 各子模块顺序固定

| 工作流包 | 固定顺序 |
|---------|---------|
| 备课包 | 教案 → PPT课件 → 课堂练习 |
| 家校沟通包 | 通知草稿 → 反馈草稿 → 沟通建议 |
| 辅导包 | 错因分析 → 分步讲解 → 巩固练习 |
| 轻量课堂包 | 简版教案 → 精简PPT → 少量练习 |

### 模块过渡语统一

模块之间使用简短过渡语：

```markdown
---

## 二、{模块名称}

以下是配套{模块名称}内容：
```

**禁止**：
- ❌ 冗长的过渡说明
- ❌ 每个模块都重复"以下是..."

### 结尾规范

工作流包输出使用统一结尾：

```markdown
---
*本{包类型}由教师助手生成，包含{模块列表}，可根据实际需要调整使用。*
```

**示例**：
```markdown
---
*本备课包由教师助手生成，包含教案、PPT、练习三部分，可根据实际需要调整使用。*
```

**禁止**：
- ❌ 每个子模块都加结尾
- ❌ 过长的使用说明
- ❌ 重复的"如需我继续..."

### 文件命名规范

工作流包文件命名格式：

```
{包类型}_{主题}_{年级}_{日期}.md
```

**示例**：
- `备课包_春_初二_20260301.md`
- `辅导包_一元二次方程_初三_20260301.md`
- `家校沟通包_期中反馈_20260301.md`

## 边界规则

### Use when

- 用户请求包含多个模块组合（如"备课+PPT+练习"）
- 用户说"备课包"、"完整备课"、"一整套课"
- 用户说"家校沟通包"、"辅导包"
- 用户说"讲解 + 练习"组合

### Do not use when

- 用户明确只要单一产物（只要教案、只要PPT等）
- 用户请求的是单独行政文案
- 用户请求的是单独辅导（无练习需求）

### 冲突让位规则

| 冲突对象 | 让位条件 | 让位方向 |
|---------|---------|---------|
| lesson-planner | 单独教案请求 | 让位给 lesson-planner |
| presentation | 单独PPT请求 | 让位给 presentation |
| admin | 单独行政请求 | 让位给 admin |
| tutor | 单独讲解请求（无练习） | 让位给 tutor |

### 边界判断示例

| 用户请求 | 路由目标 | 判断依据 |
|---------|---------|---------|
| "给我备一节课" | lesson-planner | 单独备课 |
| "给我准备一整套课" | workflow-pack | 组合型请求 |
| "备课 + PPT" | workflow-pack | 多模块组合 |
| "帮我讲这道题" | tutor | 单独讲解 |
| "讲解 + 出类似题" | workflow-pack | 组合请求 |

## 推荐模型档位

### 默认档位

| 属性 | 值 |
|-----|-----|
| 默认档位 | strong（强模型） |
| 可升级 | 不允许（已为最高档位） |
| 可降级 | standard |
| 不推荐 | light |

### 默认强模型原因

**workflow-pack 默认使用强模型**，原因：

1. 组合编排需要：多 Skill 编排需要强推理能力
2. 一致性保障：整包输出需要高度统一的风格
3. 长文本整合：整包输出通常较长，需要稳定输出
4. 规划能力：需要规划执行顺序和上下文传递

### 降级条件

满足以下条件时可降级到标准模型：

| 触发场景 | 判断标准 |
|----------|----------|
| 单一简单任务组合 | 任务极其简单 |
| 输出长度极短 | 整包输出预计 <1000字 |
| 用户明确要求快速 | 用户要求快速输出 |

### 不推荐轻量模型说明

**workflow-pack 不推荐使用轻量模型**，原因：

1. 编排能力不足：轻量模型难以处理多模块协调
2. 风格一致性差：轻量模型难以保障整包风格统一
3. 上下文理解弱：轻量模型难以理解模块间关系

### 档位选择示例

| 用户请求 | 推荐档位 | 选择原因 |
|---------|---------|---------|
| "给我准备一整套课" | strong | 组合编排，使用默认档位 |
| "备课包，要完整详细" | strong | 复杂度高，保持强模型 |
| "快速做个简版备课包" | standard | 简版输出，可降级 |
| "完整备课包，高质量" | strong | 高质量要求，保持强模型 |
