# SKILL.md - Presentation（PPT成品内容技能）

## 技能名称
presentation

## 技能描述
专门负责生成PPT成品内容包的子技能，根据教案摘要或课程信息、教师偏好，输出可直接用于制作课件的逐页内容，每页包含标题、核心内容、展示建议和教师讲解备注。

## 触发条件
由 head-teacher 技能调用，当用户请求PPT、课件、幻灯片、讲课课件等内容时触发。

## 教师偏好读取

### 可参考偏好

| 偏好项 | 来源 | 影响内容 |
|-------|------|---------|
| PPT风格 | profiles/teacher-profile.md | 简洁/重点突出/课堂互动型 |
| 文字密度偏好 | profiles/teacher-profile.md | 每页文字量 |
| 是否偏好互动提示 | profiles/teacher-profile.md | 互动页设计 |
| 每页内容量 | profiles/teacher-profile.md | 页面数量和信息密度 |

### 可参考班级画像

| 画像项 | 来源 | 影响内容 |
|-------|------|---------|
| 课堂参与度 | profiles/classroom-profile.md | 互动页设计频率 |
| 互动频率偏好 | profiles/classroom-profile.md | 提问页、活动页数量 |

### 偏好应用规则

1. **优先级**：当前任务明确要求 > 教师偏好 > 班级画像 > 通用默认规则
2. **风格适配**：
   - 偏好简洁 → 减少每页文字量，更聚焦要点
   - 偏好重点突出 → 强化总结页与关键点呈现
   - 偏好互动 → 增加提问页、互动提示
3. **班级适配**：
   - 班级参与度高 → 多设计互动环节
   - 班级参与度一般 → 适度互动，以讲授为主
4. **优雅降级**：偏好缺失时使用通用规则

### 应用示例

| 偏好组合 | 自动调整 |
|---------|---------|
| 简洁风格 + 字少图多 | 每页不超过3-5行，强调视觉 |
| 互动型 + 班级参与度高 | 增加提问页、讨论页 |
| 重点突出 + 高中学段 | 强化知识框架、方法总结页 |

## 学段适配规则

### 输入参数

| 参数名 | 必需 | 说明 |
|-------|------|------|
| 学段 | 是 | 小学/初中/高中 |

### 小学课件风格

| 维度 | 要求 |
|-----|------|
| 文字密度 | 每页不超过3-5行 |
| 句子长度 | 短句为主，每句不超过15字 |
| 配色风格 | 明亮、活泼、多彩 |
| 图片使用 | 大图为主，吸引注意力 |
| 动画效果 | 可适当丰富，增加趣味 |
| 互动设计 | 提问、游戏、操作环节多 |
| 页面节奏 | 一页一个重点，节奏快 |

**禁忌**：
- ❌ 满页文字
- ❌ 抽象概念过多
- ❌ 长篇讲解

### 初中课件风格

| 维度 | 要求 |
|-----|------|
| 文字密度 | 每页5-8行 |
| 句子长度 | 适中，表达完整 |
| 配色风格 | 清爽、专业、统一 |
| 图片使用 | 图文结合，辅助理解 |
| 动画效果 | 适度，服务于内容 |
| 互动设计 | 有提问、讨论、练习 |
| 页面节奏 | 知识点+例题+练习 |

**特点**：
- 结构清晰
- 重点突出
- 典型例题

### 高中课件风格

| 维度 | 要求 |
|-----|------|
| 文字密度 | 可适当提高，但保持清晰 |
| 句子长度 | 完整表达，逻辑清晰 |
| 配色风格 | 简洁、专业、沉稳 |
| 图片使用 | 图表、思维导图为主 |
| 动画效果 | 简洁，突出逻辑 |
| 互动设计 | 问题探究、思路分析 |
| 页面节奏 | 知识框架+方法总结+题型归纳 |

**禁忌**：
- ❌ 过度卡通化
- ❌ 过于简单幼稚
- ❌ 缺乏深度

### 学段适配检查

| 学段 | 检查要点 |
|-----|---------|
| 小学 | 是否满页文字？是否过于抽象？是否有趣？ |
| 初中 | 是否结构清晰？是否有典型例题？ |
| 高中 | 是否过于幼态？是否有深度？逻辑是否严密？ |

## 核心升级

**从"PPT大纲"升级为"PPT成品内容包"**：
- 不再只是生成页级结构大纲
- 默认生成逐页成品内容
- 每页包含完整的教学内容
- 可直接复制到PPT软件使用

## 生成规则

### 内容要求

1. **与教案一致**：PPT内容必须与教案核心目标保持一致
2. **重点突出**：教学重点要在多个页面中体现
3. **逻辑清晰**：页面顺序与教学流程对应
4. **内容完整**：每页都有实质内容，不只是标题
5. **讲解详尽**：教师讲解备注要具体可用

### 展示建议要求

每页必须包含具体的展示建议：
- **背景图片**：描述建议的背景类型
- **配色方案**：主色调和辅助色
- **布局方式**：内容排列方式
- **动画效果**：适当的动画建议
- **字体大小**：标题和正文的字号建议

### 讲解备注要求

教师讲解备注必须：
- 完整可读，不是碎片化的提示
- 包含开场白、过渡语、总结语
- 体现教学重点的强调
- 包含与学生的互动设计

### 与教案的对应关系

| 教案环节 | 对应PPT页面 | 内容要点 |
|---------|------------|---------|
| 导入 | 封面页 + 导入页 | 激发兴趣，引入课题 |
| 新授 | 核心知识讲解页（多页） | 知识点逐层展开 |
| 练习 | 示例/互动页 + 课堂练习页 | 巩固应用 |
| 总结 | 总结页 | 归纳提升 |
| 作业 | 作业布置页 | 延伸拓展 |

## 输出格式

- 格式：Markdown
- 语言：中文
- 风格：结构清晰、内容完整、便于直接使用

## 边界限制

本技能：
- ✅ 生成PPT成品内容包
- ✅ 提供每页完整内容
- ✅ 提供展示设计建议
- ✅ 提供教师讲解备注
- ❌ 不生成真实.pptx文件（可选功能，暂未实现）
- ❌ 不修改教案核心目标
- ❌ 不脱离教学主线乱扩展
- ❌ 不自动搜图

## 降级规则

如果无法生成完整内容：
1. 至少保证每页有标题和核心内容
2. 展示建议可简化为通用建议
3. 讲解备注可简化为要点提示
4. 不允许只输出页面标题列表

## 第2页：导入页

### 页面标题
走进春天

### 页面核心内容
- 导入方式：图片欣赏 + 问题引导
- 导入内容：
  - 欣赏春天的美景图片
  - 分享对春天的印象
- 思考问题：你眼中的春天是什么样的？

### 展示建议
- **背景**：春天美景图片轮播（春草、春花、春风、春雨）
- **布局**：左侧图片，右侧文字和问题
- **动画**：图片轮播，文字逐条显示
- **配乐**：可播放轻柔的背景音乐

### 教师讲解备注
```
首先，让我们来欣赏几幅春天的图片。

（播放图片）

看到这些图片，你们有什么感受？春天给你最深的印象是什么？

（等待学生回答，给予肯定）

同学们说得都很好。春天是美丽的、生机勃勃的。那么，朱自清先生笔下的春天又是怎样的呢？让我们一起来学习这篇课文。
```

## 第3页：核心知识讲解页（一）

### 页面标题
初读感知——文章结构

### 页面核心内容
- 文章结构：
  - 盼春（第1自然段）
  - 绘春（第2-7自然段）
    - 春草图
    - 春花图
    - 春风图
    - 春雨图
    - 迎春图
  - 赞春（第8-10自然段）

### 展示建议
- **背景**：简洁的浅色背景
- **布局**：使用结构图展示文章脉络
- **配色**：三个部分用不同颜色区分
- **动画**：结构图逐层展开

### 教师讲解备注
```
首先，我们来整体感知一下文章的结构。

这篇文章可以分为三个部分：

第一部分是"盼春"，在文章的第一自然段，作者表达了对春天的盼望之情。

第二部分是"绘春"，这是文章的主体部分，作者描绘了五幅春景图：春草图、春花图、春风图、春雨图，还有迎春图。

第三部分是"赞春"，在文章的最后三个自然段，作者用三个比喻句赞美了春天。

这样的结构，从盼春到绘春再到赞春，层次非常清晰。
```

## 第4页：核心知识讲解页（二）

### 页面标题
品味语言——修辞之美

### 页面核心内容
- 比喻句：
  - "红的像火，粉的像霞，白的像雪"
  - 作用：生动形象地写出春花的颜色多、色彩艳
- 拟人句：
  - "小草偷偷地从土里钻出来"
  - 作用：赋予小草人的情态，写出小草的可爱和生命力
- 排比句：

## 统一输出规范

### 标题格式统一

PPT输出必须使用统一的标题格式：

```markdown
# PPT成品内容包：{课题}
```

**示例**：
- `# PPT成品内容包：《春》`
- `# PPT成品内容包：一元二次方程`

### 页面标题风格统一

每页标题使用固定格式：

```markdown
## 第{N}页：{页面类型}
```

**固定页面类型命名**：
| 页面类型 | 命名 |
|---------|------|
| 封面 | 封面页 |
| 导入 | 导入页 |
| 知识讲解 | 核心知识讲解页 |
| 示例互动 | 示例/互动页 |
| 练习 | 课堂练习页 |
| 总结 | 总结页 |
| 作业 | 作业布置页 |

### 页面字段顺序固定

每页内容字段顺序固定：

```markdown
### 页面标题
{标题内容}

### 页面核心内容
{核心内容列表}

### 展示建议
{展示建议}

### 教师讲解备注
{讲解内容}
```

**禁止变体命名**：
- ❌ "展示方式" → 应为"展示建议"
- ❌ "呈现建议" → 应为"展示建议"
- ❌ "讲解词" → 应为"教师讲解备注"
- ❌ "备注" → 应为"教师讲解备注"

### 结尾规范

PPT输出使用简短结尾：

```markdown

*本PPT成品内容包由教师助手生成，可根据实际需要调整。*
```

**禁止**：
- ❌ 过长的使用说明
- ❌ 重复的"如需我继续..."

## 边界规则

### Use when

- 用户明确只要PPT或课件内容
- 用户说"给我做个PPT"且无其他组合需求
- 用户说"帮我做个课件"且未提及教案或练习

- 用户请求的是完整备课包
- 用户请求的是单独教案
- 用户请求的是单独作业或练习
- 用户请求的是行政文案或辅导
