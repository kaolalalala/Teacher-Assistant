# 模型路由配置

## 概述

本文件定义了教师助手系统中不同任务类型的默认模型分配策略，用于实现任务级模型路由。

## 模型分层定义

### 第一层：强模型（Tier-1）

**特点**：推理能力强、输出质量高、成本较高

**适用场景**：
- 复杂推理任务
- 需要深度分析的内容
- 工作流包总控
- 用户明确要求高质量输出

### 第二层：中等模型（Tier-2）

**特点**：平衡质量与成本、响应较快

**适用场景**：
- 常规生成任务
- 结构化输出
- 一般复杂度内容

### 第三层：轻量模型（Tier-3）

**特点**：成本低、响应快、适合简单任务

**适用场景**：
- 简单格式化任务
- 日志记录
- 状态回复
- 成本敏感场景

## 任务类型 → 模型映射

### 核心业务任务

| 任务类型 | 默认模型层级 | 备选模型层级 | 优先级 | 说明 |
|---------|-------------|-------------|-------|------|
| 教案生成 | Tier-1 | Tier-2 | 质量优先 | 教案是核心产出，需要较强推理能力 |
| PPT/课件 | Tier-1 | Tier-2 | 质量优先 | 需要结构化思维和创意设计 |
| 学生辅导 | Tier-1 | Tier-2 | 质量优先 | 辅导需要深度分析和启发式引导 |
| 作业/练习 | Tier-2 | Tier-3 | 平衡 | 题目生成需要一定质量，但可适度控制成本 |
| 行政文案 | Tier-3 | Tier-2 | 成本优先 | 通知、评语等相对简单，可用轻量模型 |

### 工作流包任务

| 工作流包阶段 | 默认模型层级 | 备选模型层级 | 说明 |
|------------|-------------|-------------|------|
| 备课包-总控 | Tier-1 | Tier-2 | 总控需要规划能力 |
| 备课包-教案 | Tier-1 | Tier-2 | 核心产出 |
| 备课包-PPT | Tier-1 | Tier-2 | 核心产出 |
| 备课包-练习 | Tier-2 | Tier-3 | 可适度控制成本 |
| 辅导包-分析 | Tier-1 | Tier-2 | 需要分析能力 |
| 辅导包-讲解 | Tier-1 | Tier-2 | 核心产出 |
| 辅导包-练习 | Tier-2 | Tier-3 | 可适度控制成本 |
| 家校沟通包-通知 | Tier-3 | Tier-2 | 相对简单 |
| 家校沟通包-反馈 | Tier-2 | Tier-3 | 需要一定质量 |

### 系统任务

| 任务类型 | 默认模型层级 | 备选模型层级 | 优先级 | 说明 |
|---------|-------------|-------------|-------|------|
| 输出整形 | Tier-2 | Tier-3 | 平衡 | 格式化和整合任务 |
| 质量治理 | Tier-2 | Tier-3 | 平衡 | 检查和修正任务 |
| 观察记录 | Tier-3 | - | 成本优先 | 简单日志记录 |
| 评测结果整理 | Tier-3 | - | 成本优先 | 简单汇总任务 |
| 模式切换 | Tier-3 | - | 成本优先 | 简单状态操作 |

## 降级策略

### 降级触发条件

1. 目标模型不可用
2. 目标模型响应超时
3. 目标模型负载过高
4. 用户明确要求使用其他模型

### 降级规则

```
IF 目标模型不可用
   → 尝试备选模型
   IF 备选模型也不可用
      → 使用当前可用主模型
      → 记录降级事件到日志
   END
END

确保：不因模型路由失败而中断主流程
```

### 降级记录

降级事件应记录到 `logs/errors/` 目录，包含：

- 原始目标模型
- 降级原因
- 最终使用模型
- 任务类型
- 时间戳

## 特殊场景路由

### 用户明确指定模型

当用户明确要求使用特定模型或风格时：

```
用户明确要求 > 默认路由规则
```

例如：
- "用最强模型给我写一份教案" → 强制使用 Tier-1
- "快速生成一份通知" → 可使用 Tier-3

### 高负载场景

当系统负载较高时，可临时调整路由策略：

| 场景 | 调整策略 |
|-----|---------|
| 高负载 | 所有任务降一级 |
| 极高负载 | 仅使用轻量模型 |
| 正常 | 按默认规则执行 |

### 实验模式

当需要进行模型对比实验时：

1. 记录实验配置
2. 同一任务使用不同模型执行
3. 记录结果差异
4. 写入评测结果文件

## 模型路由优先级

```
1. 用户明确指定模型
2. 任务类型默认映射
3. 系统负载状态调整
4. 备选模型降级
5. 兜底：使用当前可用主模型
```

---

## 注意事项

1. **路由是"按任务类型"而不是"随机选择"**
2. **默认映射必须可读、可修改**
3. **后续评测应能观察模型变更是否带来回归退化**
4. **模型路由不应改变业务规则本身**
5. **即使当前只有 1~2 个模型，也要保留这套逻辑分层，方便后续替换**
