# 工作流包模板

> 本模板定义工作流包的标准结构，供系统识别和执行套餐化输出。

---

## 概述

工作流包是"组合调度"能力，将多个基础模块封装成可复用的高频套餐。每个工作流包定义了：

- 适用场景和触发词
- 包含的子模块
- 默认执行顺序
- 降级策略
- 输出规范

---

## 一、备课包（Lesson Pack）

### 适用场景

- 完整备课、一整套课、备课包
- 备课 + PPT + 练习
- 教学整包、课堂整包

### 触发词

| 关键词组合 | 说明 |
|-----------|------|
| 备课包 | 直接触发 |
| 完整备课 | 直接触发 |
| 一整套课 | 直接触发 |
| 备课 + PPT + 练习 | 直接触发 |
| 教学整包 | 直接触发 |
| 课堂整包 | 直接触发 |

### 默认包含模块

| 序号 | 模块 | 说明 |
|-----|------|------|
| 1 | 教案 | 结构完整的教学设计 |
| 2 | PPT/课件 | 逐页成品内容包 |
| 3 | 练习/作业 | 默认为课堂练习（4-6题） |

### 执行顺序

```
lesson-planner → presentation → assignment
```

### 输出顺序

1. 教案
2. PPT/课件
3. 配套练习

### 降级策略

| 信息缺失 | 降级处理 |
|---------|---------|
| 学科不明 | 默认语文 |
| 年级不明 | 默认初中 |
| 课题不明 | 询问用户或生成通用模板 |
| 课时不明 | 默认40分钟 |

### 裁剪规则

| 用户要求 | 裁剪方式 |
|---------|---------|
| 不要PPT | 只输出教案 + 练习 |
| 不要练习 | 只输出教案 + PPT |
| 只要教案 | 只输出教案 |
| 课后作业 | 将练习改为课后作业格式 |

## 五、工作流包执行规则

### 优先级规则

工作流包内各模块仍然遵循：

```
当前用户明确要求 > 教师偏好 > 班级层面画像 > 通用默认规则
```

### 风格统一要求

- 所有子产物使用相同的学段风格
- 所有子产物使用相同的语言风格
| 输出格式保持一致
- 避免重复内容

### 验收规则

主 agent 必须检查：

| 检查项 | 要求 |
|-------|------|
| 完整性 | 所有应有子产物都已生成 |
| 顺序性 | 输出顺序符合定义 |
| 一致性 | 不同模块语气和风格统一 |
| 无冲突 | 不存在明显重复或冲突 |
| 适配性 | 符合对应学段与教师偏好 |

### 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 子模块失败 | 尝试重新执行或跳过，继续其他模块 |
| 信息不足 | 使用降级策略，不直接失败 |
| 用户裁剪 | 尊重用户要求，调整工作流内容 |
