# SKILL.md - Model Router（模型路由技能）

## 技能名称

model-router

## 技能描述

模型路由层技能，负责根据任务类型选择合适的模型、管理模型降级、提供路由决策支持。该技能不替代业务技能，而是为系统提供模型调度能力。

## 触发条件

- 执行单模块任务时
- 执行工作流包时
- 执行评测时
- 做多模型效果对比时
- 需要控制成本或保证质量时

## 核心职责

1. **模型选择**：根据任务类型选择合适的模型层级
2. **降级管理**：处理模型不可用时的降级逻辑
3. **路由决策**：提供模型路由的决策支持
4. **成本控制**：在保证质量的前提下优化成本

## 模型分层定义

### Tier-1：强模型

**特点**：
- 推理能力强
- 输出质量高
- 成本较高

**适用场景**：
- 复杂推理任务
- 需要深度分析的内容
- 工作流包总控
- 用户明确要求高质量输出

### Tier-2：中等模型

**特点**：
- 平衡质量与成本
- 响应较快
- 适合常规任务

**适用场景**：
- 常规生成任务
- 结构化输出
- 一般复杂度内容

### Tier-3：轻量模型

**特点**：
- 成本低
- 响应快
- 适合简单任务

**适用场景**：
- 简单格式化任务
- 日志记录
- 状态回复
- 成本敏感场景

## 默认路由策略

### 高质量优先任务

建议使用 Tier-1（强模型）：

| 任务类型 | 说明 |
|---------|------|
| 教案生成 | 需要深度教学设计能力 |
| PPT/课件 | 需要结构化思维和创意设计 |
| 学生辅导 | 需要深度分析和启发式引导 |
| 工作流包总控 | 需要规划和协调能力 |

### 平衡型任务

建议使用 Tier-2（中等模型）：

| 任务类型 | 说明 |
|---------|------|
| 作业/练习生成 | 需要一定质量，可适度控制成本 |
| 输出整形 | 格式化和整合任务 |
| 一般辅导 | 常规讲解任务 |
| 常规总结 | 汇总类任务 |

### 成本优先任务

建议使用 Tier-3（轻量模型）：

| 任务类型 | 说明 |
|---------|------|
| 行政文案 | 通知、评语等相对简单 |
| 观察日志 | 简单记录任务 |
| 评测结果整理 | 简单汇总任务 |
| 简单状态回复 | 确认类回复 |

## 路由决策流程

```
步骤1：接收任务类型
   → 从主 agent 获取任务类型

步骤2：读取路由配置
   → 读取 profiles/model-routing.md
   → 获取默认模型层级和备选模型层级

步骤3：检查用户要求
   IF 用户明确指定模型
      → 返回用户指定的模型层级
   ELSE
      → 继续默认流程

步骤4：检查系统状态
   → 检查目标模型是否可用
   → 检查系统负载状态

步骤5：返回路由决策
   → 返回推荐的模型层级
   → 返回备选模型层级
   → 返回降级策略
```

## 降级逻辑

### 降级触发条件

| 条件 | 说明 |
|-----|------|
| 模型不可用 | 目标模型暂时无法访问 |
| 响应超时 | 目标模型响应时间过长 |
| 负载过高 | 目标模型负载过高 |
| 用户要求 | 用户明确要求使用其他模型 |

### 降级规则

```
IF 目标模型不可用
   → 尝试备选模型
   IF 备选模型也不可用
      → 使用当前可用主模型
      → 记录降级事件到日志
   END
END

确保：不因模型路由失败而中断主流程
```

### 降级记录

降级事件应记录到 `logs/errors/` 目录：

```markdown
# 模型降级记录

| 字段 | 内容 |
|-----|------|
| 时间 | YYYY-MM-DD HH:mm:ss |
| 任务类型 | [任务类型] |
| 原始目标模型 | Tier-X |
| 降级原因 | [原因] |
| 最终使用模型 | Tier-Y |
| 影响 | [影响说明] |
```

## 特殊场景处理

### 用户明确指定模型

当用户明确要求使用特定模型时：

```
用户明确要求 > 默认路由规则
```

识别用户要求的模式：

| 用户表述 | 识别为 |
|---------|-------|
| "用最强模型"、"高质量"、"仔细分析" | Tier-1 |
| "正常就行"、"标准输出" | Tier-2 |
| "快速生成"、"简单处理"、"不用太复杂" | Tier-3 |

### 高负载场景

当系统负载较高时，可临时调整路由策略：

| 场景 | 调整策略 |
|-----|---------|
| 高负载 | 所有任务降一级 |
| 极高负载 | 仅使用 Tier-3 |
| 正常 | 按默认规则执行 |

### 实验模式

当需要进行模型对比实验时：

1. 记录实验配置到 `logs/`
2. 同一任务使用不同模型执行
3. 记录结果差异
4. 写入评测结果文件

## 路由优先级

```
1. 用户明确指定模型
2. 任务类型默认映射
3. 系统负载状态调整
4. 备选模型降级
5. 兜底：使用当前可用主模型
```

## 注意事项

1. **不替代业务技能**：本技能用于模型调度，不直接生成业务输出
2. **保持可读性**：路由规则必须可读、可修改
3. **优雅降级**：模型不可用时不中断主流程
4. **记录可追溯**：降级事件必须记录
5. **兼容单模型**：即使只有 1~2 个模型，也要保留逻辑分层
