# SKILL.md - Lesson Planner（教案技能）

## 技能名称
lesson-planner

## 技能描述
专门负责生成结构化教案的子技能，根据学科、年级、课题、学段、教师偏好等信息，输出符合学段特点和个人风格的教学设计方案。

## 触发条件
由 head-teacher 技能调用，当用户请求备课、教案、课堂设计、教学设计时触发。

## 教师偏好读取

### 可参考偏好

| 偏好项 | 来源 | 影响内容 |
|-------|------|---------|
| 教案风格 | profiles/teacher-profile.md | 简洁/详细/活动型/重点突破型 |
| 是否偏爱探究式教学 | profiles/teacher-profile.md | 教学活动设计 |
| 是否更重练习设计 | profiles/teacher-profile.md | 作业设计部分 |
| 是否偏重考试导向 | profiles/teacher-profile.md | 教学目标和重点设计 |
| 教案语言风格 | profiles/teacher-profile.md | 口语化/规范化/学术化 |

### 可参考班级画像

| 画像项 | 来源 | 影响内容 |
|-------|------|---------|
| 整体基础水平 | profiles/classroom-profile.md | 教学难度和深度 |
| 教学节奏偏好 | profiles/classroom-profile.md | 环节时长分配 |
| 课堂互动偏好 | profiles/classroom-profile.md | 活动设计频率 |

### 偏好应用规则

1. **优先级**：当前任务明确要求 > 教师偏好 > 班级画像 > 通用默认规则
2. **风格适配**：
   - 偏好简洁 → 教案更短、更聚焦核心
   - 偏好详细 → 步骤完整、细节丰富
   - 偏好活动型 → 增加互动环节设计
   - 偏好重点突破 → 更强调重难点设计
3. **班级适配**：
   - 班级基础偏弱 → 降低难度、增加基础环节
   - 班级基础较强 → 增加拓展内容、提高思维要求
4. **优雅降级**：偏好缺失时使用通用规则

### 应用示例

| 偏好组合 | 自动调整 |
|---------|---------|
| 简洁教案 + 班级基础偏弱 | 精简结构，聚焦基础，步骤更细 |
| 活动型 + 班级互动偏好高 | 多设计小组活动、讨论环节 |
| 考试导向 + 班级基础较强 | 增加考点分析、典型例题 |

## 学段适配规则

### 输入参数

| 参数名 | 必需 | 说明 |
|-------|------|------|
| 学段 | 是 | 小学/初中/高中 |
| 学科 | 是 | 教学学科 |
| 年级 | 是 | 教学年级 |
| 课题 | 是 | 课程主题 |

### 小学教案风格

| 维度 | 要求 |
|-----|------|
| 整体风格 | 活泼、趣味、生活化 |
| 语言表达 | 口语化、简单易懂、多用比喻 |
| 导入设计 | 故事、游戏、情境导入为主 |
| 活动设计 | 具体可操作、时间短、互动多 |
| 环节时长 | 每环节5-10分钟，节奏快 |
| 板书设计 | 图文并茂、色彩丰富 |
| 评价方式 | 鼓励性为主，多用表扬 |
| 作业设计 | 少量、有趣、实践性强 |

**示例特点**：
- 教学目标表述简单直接
- 教学流程多用"小朋友们"
- 活动设计具体到"谁做什么说什么"
- 强调"趣味性"和"参与度"

### 初中教案风格

| 维度 | 要求 |
|-----|------|
| 整体风格 | 结构清晰、逻辑性强 |
| 语言表达 | 规范准确、适度专业 |
| 导入设计 | 问题导入、情境导入、复习导入 |
| 活动设计 | 有明确目标、时间适中 |
| 环节时长 | 每环节10-15分钟，节奏适中 |
| 板书设计 | 条理清晰、重点突出 |
| 评价方式 | 过程与结果并重 |
| 作业设计 | 分层设计、典型题目 |

**示例特点**：
- 教学目标分三维表述
- 教学流程结构化
- 重难点明确
- 强调"理解"和"应用"

### 高中教案风格

| 维度 | 要求 |
|-----|------|
| 整体风格 | 紧凑高效、深度分析 |
| 语言表达 | 专业规范、逻辑严密 |
| 导入设计 | 直奔主题、问题驱动 |
| 活动设计 | 思维训练为主、探究性强 |
| 环节时长 | 每环节15-20分钟，深度展开 |
| 板书设计 | 知识框架、思维导图 |
| 评价方式 | 能力导向、方法评价 |
| 作业设计 | 综合性强、有区分度 |

**示例特点**：
- 教学目标聚焦核心素养
- 教学流程强调思维过程
- 重难点分析深入
- 强调"迁移"和"方法"

### 学段适配检查

生成教案后必须检查：

| 学段 | 检查要点 |
|-----|---------|
| 小学 | 是否活泼有趣？环节是否短小？语言是否简单？ |
| 初中 | 是否结构清晰？重难点是否明确？难度是否适中？ |
| 高中 | 是否有深度？逻辑是否严密？是否过于幼态？ |

## 输出保存

### 保存规则

1. **自动保存**：每次生成教案后，必须保存到文件
2. **保存路径**：`outputs/lessons/`
3. **命名规则**：`{学科}_{年级}_{课题}_{日期}.md`
4. **日期格式**：YYYYMMDD

### 命名示例

```

数学_高一_函数的概念_20260228.md
英语_初三_现在完成时_20260228.md
```

### 文件冲突处理

- 如文件已存在，添加序号后缀：`_2`、`_3`...
- 示例：`语文_初二_春_20260228_2.md`

## 生成规则

### 内容要求

1. **符合课程标准**：内容需符合对应年级学科的课程标准
2. **目标明确具体**：教学目标可测量、可达成
3. **流程合理**：时间分配合理，环节衔接自然
4. **活动有效**：师生活动设计有针对性
5. **作业适量**：作业量适中，有层次性

### 时间分配建议

| 课时 | 导入 | 新授 | 练习 | 总结 |
|-----|------|------|------|------|
| 40分钟 | 3-5分钟 | 20-25分钟 | 8-10分钟 | 2-3分钟 |
| 45分钟 | 3-5分钟 | 25-30分钟 | 10-12分钟 | 2-3分钟 |

### 学科特色

**语文**：
- 注重朗读、品味语言
- 强调文本理解与鉴赏
- 关注写作手法分析

**数学**：
- 注重概念理解
- 强调解题方法
- 关注逻辑思维培养

**英语**：
- 注重听说读写综合训练
- 强调词汇和语法
- 关注语言运用能力

**物理/化学**：
- 注重实验探究
- 强调概念理解
- 关注科学思维培养

**历史/地理**：
- 注重时空观念
- 强调史料分析
- 关注人文素养

## 输出格式

- 格式：Markdown
- 语言：中文
- 风格：结构清晰、直接可用

## 边界限制

本技能：
- ✅ 生成结构化教案
- ✅ 补全缺失的教学目标
- ✅ 设计教学流程和活动
- ❌ 不生成PPT大纲
- ❌ 不扩展到科研、行政等任务
- ❌ 不生成真实课件文件

## 示例

**输入**：
```
学科：语文
年级：初二
课题：《春》
课时：40分钟
```

**输出**：
```markdown
# 教案：《春》

## 一、课题信息

- **学科**：语文
- **年级**：初二
- **课题**：《春》（朱自清）
- **课时**：1课时（40分钟）
- **课型**：新授课

## 二、教学目标

### 知识与技能
1. 掌握生字词，理解课文内容
2. 学习比喻、拟人等修辞手法的运用
3. 背诵精彩段落

### 过程与方法
1. 通过朗读品味优美语言
2. 学习抓住景物特征描写的方法
3. 培养观察和表达能力

### 情感态度与价值观
1. 感受春天的美好，培养热爱自然的情感
2. 体会作者对春天的赞美之情

## 三、教学重难点

### 教学重点
1. 品味优美语言，学习修辞手法
2. 理解课文内容，感受春天之美

### 教学难点
1. 学习抓住景物特征进行描写的方法
2. 体会作者在景物描写中抒发的情感

## 四、教学流程

### 环节一：导入（约4分钟）
播放春天的图片或视频，引导学生说说对春天的印象，引出课题。

### 环节二：初读感知（约8分钟）
1. 学生自由朗读课文，圈画生字词
2. 检查预习，正音正字
3. 概括课文主要内容，理清文章结构

### 环节三：精读品析（约18分钟）
1. 分组研读：春草、春花、春风、春雨、迎春五幅图景
2. 品味语言：找出喜欢的句子，分析修辞手法
3. 朗读指导：有感情地朗读精彩段落

### 环节四：总结拓展（约6分钟）
1. 总结写作方法：抓住特征、运用修辞、情景交融
2. 小练笔：仿写一个季节的片段

### 环节五：布置作业（约4分钟）
布置分层作业，预告下节课内容。

## 五、师生活动与板书建议

### 师生活动设计
| 教师活动 | 学生活动 | 设计意图 |
|---------|---------|---------|
| 播放视频，引导观察 | 观看视频，分享感受 | 激发兴趣，导入新课 |
| 示范朗读，指导方法 | 模仿朗读，体会情感 | 培养语感，感受语言美 |
| 提问引导，组织讨论 | 小组讨论，代表发言 | 培养合作学习能力 |

### 板书设计
```
                    《春》 朱自清
                    
    春草：嫩、绿、多、软
    春花：争芳斗艳        抓住特征
    春风：柔和、芳香      运用修辞
    春雨：细、密、亮      情景交融
    迎春：人勤春早
    
    赞美春天 → 热爱自然
```

## 六、作业设计

### 必做题
1. 抄写并掌握本课生字词
2. 背诵课文第2-5自然段

### 选做题
1. 仿照课文写法，描写一个你喜欢的季节
2. 搜集描写春天的古诗词，下节课分享
```

## 边界规则

### Use when

- 用户明确只要教案或备课内容
- 用户请求单独的教学设计
- 用户说"给我备一节课"且无其他组合需求
- 用户说"帮我做个教案"且未提及PPT或练习

### Do not use when

- 用户请求的是完整备课包（含PPT/练习）
- 用户请求的是PPT或课件
- 用户请求的是作业或练习
- 用户请求的是行政文案或辅导

### 冲突让位规则

| 冲突对象 | 让位条件 | 让位方向 |
|---------|---------|---------|
| workflow-pack | 组合型请求（备课+PPT+练习） | 让位给 workflow-pack |
| presentation | 同时包含教案和PPT | 让位给 workflow-pack |
| assignment | 同时包含教案和练习 | 让位给 workflow-pack |

### 边界判断示例

| 用户请求 | 路由目标 | 判断依据 |
|---------|---------|---------|
| "给我备一节课" | lesson-planner | 单独备课 |
| "帮我做个教案" | lesson-planner | 单独教案 |
| "给我准备一整套课" | workflow-pack | 组合型请求 |
| "备课 + PPT" | workflow-pack | 多模块组合 |

## 推荐模型档位

### 默认档位

| 属性 | 值 |
|-----|-----|
| 默认档位 | standard（标准模型） |
| 可升级 | strong |
| 可降级 | light |
| 不推荐 | 无 |

### 升级条件

满足以下条件时升级到强模型：

| 触发场景 | 判断标准 |
|----------|----------|
| 详细教案 | 完整教案（含目标、过程、评价） |
| 长篇教学设计 | 输出预计 >2000字 |
| 跨学科整合 | 跨学科/跨课时整合设计 |
| 强一致性要求 | 需要高度统一的风格和格式 |

### 降级条件

满足以下条件时可降级到轻量模型：

| 触发场景 | 判断标准 |
|----------|----------|
| 简版教案 | 输出预计 <500字 |
| 大纲级输出 | 仅需框架结构 |
| 快速备课 | 用户明确要求快速输出 |

### 档位选择示例

| 用户请求 | 推荐档位 | 选择原因 |
|---------|---------|---------|
| "帮我备一节《春》的课" | standard | 标准复杂度 |
| "设计一份完整的跨学科教学设计" | strong | 跨学科整合，复杂度高 |
| "快速做个教案大纲" | light | 大纲级，复杂度低 |
| "详细教案，要完整的" | strong | 明确要求详细 |
