# SKILL.md - Quality Governor（质量治理技能）

## 技能名称
quality-governor

## 技能描述
质量治理技能是"输出治理层"，负责判断输出是否统一、如何做最终格式整合、如何处理多模块拼接时的重复/冲突/风格割裂，以及如何统一摘要、标题、结尾和文件命名。

该技能不替代基础技能，而是为主 agent 提供质量治理规则和检查标准。

## 触发条件

当出现以下情况时使用本技能：

| 场景 | 说明 |
|-----|------|
| 多模块输出 | 输出来自多个模块（如教案+PPT+练习） |
| 工作流包结果 | 备课包、辅导包、家校沟通包等 |
| 用户要求规范 | 用户希望"更规范/更整洁/更正式" |
| 整包内容 | 同一结果包含多个子部分 |

## 治理内容

### 1. 统一标题样式

**检查项**：
- 一级标题是否为任务结果名称
- 二级标题是否为模块名称
- 标题格式是否统一

**规范格式**：
```
# {类型}：{主题}

## 一、{模块名称}

### {子部分名称}
```

**示例**：
```markdown
# 教案：《春》

## 一、教学目标
...

## 二、教学重难点
...
```

### 2. 统一小节层级

**检查项**：
- H1/H2/H3 层级是否正确
- 是否有层级跳跃（如H1直接到H3）
- 小节命名是否稳定

**规范要求**：
| 层级 | 用途 | 示例 |
|-----|------|------|
| H1 | 任务结果名称 | `# 教案：《春》` |
| H2 | 模块名称或主要部分 | `## 教学目标` |
| H3 | 子部分或具体环节 | `### 导入环节` |
| H4 | 细节说明（谨慎使用） | `#### 设计意图` |

### 3. 统一用词风格

**检查项**：
- 语气是否一致（专业/清晰/可用）
- 是否有语气割裂
- 是否有过度口语化或过度书面化

**统一语气**：
- 专业：用词准确，避免口语化随意表达
- 清晰：结构分明，要点突出
- 可用：老师可直接使用或稍作修改即可使用

**禁止**：
- ❌ 过度口语化（如"咱们"、"吧"、"呢"过多）
- ❌ 过度书面化（如生僻词堆砌）
- ❌ 机器感（如"以下是生成的..."）
- ❌ 重复啰嗦（如反复"请注意"）

### 4. 去重

**检查项**：
- 各模块间是否有重复说明
- 是否有重复的提示语
- 是否有重复的结尾

**去重规则**：
| 重复类型 | 处理方式 |
|---------|---------|
| 说明文字重复 | 保留首次出现，删除后续 |
| 提示语重复 | 合并为一个提示 |
| 结尾重复 | 整包只需一个结尾 |

### 5. 增加简短导读

**触发条件**：
- 输出内容超过 100 行
- 来自多个模块的整包输出
- 工作流包结果

**格式**：
```markdown
> **导读**：本{类型}包含{内容概要}，共{数量}部分。
```

**示例**：
```markdown
> **导读**：本备课包包含教案、PPT课件、课堂练习三部分，已按初中风格生成。
```

### 6. 控制结尾长度

**规范**：
- 单模块结尾不超过 2 行
- 多模块整包只需一个结尾
- 不要每个模块都加结尾

**单模块结尾**：
```markdown
---
*本内容由教师助手生成，可根据实际需要调整。*
```

**多模块结尾**：
```markdown
---
*本备课包由教师助手生成，包含教案、PPT、练习三部分，可根据实际需要调整使用。*
```

### 7. 规范文件命名

**命名结构**：
```
{类型}_{主题}_{学段/年级}_{日期}.{扩展名}
```

**命名速查表**：
| 类型 | 格式 | 示例 |
|-----|------|------|
| 教案 | `教案_{课题}_{年级}_{日期}.md` | `教案_春_初二_20260301.md` |
| PPT大纲 | `PPT大纲_{课题}_{年级}_{日期}.md` | `PPT大纲_春_初二_20260301.md` |
| PPT成品 | `PPT成品_{课题}_{年级}_{日期}.md` | `PPT成品_春_初二_20260301.md` |
| 练习 | `练习_{课题}_{年级}_{日期}.md` | `练习_春_初二_20260301.md` |
| 辅导 | `辅导_{知识点}_{年级}_{日期}.md` | `辅导_一元二次方程_初三_20260301.md` |
| 行政文案 | `{类型}_{对象}_{日期}.md` | `家长会通知_家长_20260301.md` |
| 备课包 | `备课包_{课题}_{年级}_{日期}.md` | `备课包_春_初二_20260301.md` |

## 治理流程

```
步骤1：收集所有子模块输出

步骤2：标题检查
   → 统一为 "# {类型}：{主题}" 格式
   → 确保 H2 为模块名，H3 为子部分

步骤3：层级检查
   → 检查是否有层级跳跃
   → 确保层级正确

步骤4：去重处理
   → 删除重复的说明文字
   → 合并相似的提示语
   → 删除重复的结尾

步骤5：语气检查
   → 确保语气一致
   → 删除过度口语化或书面化内容

步骤6：添加导读（如适用）
   → 输出较长或来自多模块时添加

步骤7：统一结尾
   → 整包只需一个结尾

步骤8：文件命名检查
   → 确保命名符合规范

步骤9：最终输出
```

## 验收标准

| 检查项 | 标准 |
|-------|------|
| 标题统一 | 所有模块标题风格一致 |
| 层级一致 | 小节层级正确，无跳跃 |
| 内容去重 | 无重复说明、提示、结尾 |
| 语气连贯 | 整体语气专业、清晰、可用 |
| 长度适中 | 无过长、过短、风格失衡 |
| 导读规范 | 多模块输出有简短导读 |
| 结尾统一 | 整包只有一个结尾 |
| 命名规范 | 文件命名符合规范 |

## 边界规则

### Use when

- 主任务完成后需要对输出进行整形
- 多模块输出需要整合与去重
- 用户要求"更规范/更整洁/更正式"

### Do not use when

- 首轮路由阶段
- 作为主路由目标
- 抢占业务类 skill

### 冲突让位规则

本技能为**后处理类**，不得抢占首轮主路由：

| 场景 | 行为 |
|-----|------|
| 首轮路由 | 不参与，让位给业务类 skill |
| 主任务完成后 | 可参与，进行输出整形 |

### 后处理约束

- ❌ 不得作为首轮路由目标
- ❌ 不得抢占业务类 skill
- ❌ 不得在主任务未完成时介入
- ✅ 只在主任务完成后作为后置步骤参与

## 推荐模型档位

### 默认档位

| 属性 | 值 |
|-----|-----|
| 默认档位 | light（轻量模型） |
| 可升级 | standard |
| 可降级 | 不允许（已为最低档位） |
| 不推荐 | strong |

### 默认轻量模型原因

**quality-governor 默认使用轻量模型**，原因：

1. 整形任务简单：格式整理、去重、标题统一等任务复杂度低
2. 成本控制优先：后处理任务不应占用高档模型
3. 质量已可保障：轻量模型足以完成整形任务

### 升级条件

满足以下条件时升级到标准模型：

| 触发场景 | 判断标准 |
|----------|----------|
| 复杂格式整形 | 多维度格式调整 |
| 多维度质量检查 | 需要语义理解的质量检查 |
| 需要语义理解 | 整形需要理解内容语义 |

### 不推荐强模型说明

**quality-governor 不推荐使用强模型**，原因：

1. 整形任务复杂度有限：格式整理不需要强推理能力
2. 成本效益低：强模型用于整形任务性价比极低
3. 响应时间：后处理不应增加过多延迟

### 档位选择示例

| 场景 | 推荐档位 | 选择原因 |
|---------|---------|---------|
| 常规输出整形 | light | 标准复杂度，使用默认档位 |
| 复杂格式调整 | standard | 复杂整形，升级标准模型 |
| 简单去重整理 | light | 简单任务，保持轻量 |
| 多模块深度整合 | standard | 需要语义理解，升级标准模型 |
