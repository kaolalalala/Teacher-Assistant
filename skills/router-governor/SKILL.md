# SKILL.md - Router Governor（路由治理技能）

## 技能名称

router-governor

## 技能描述

路由治理层技能，负责统一判断请求属于哪一类、处理 skill 边界重叠、按冲突矩阵裁决、按统一兜底策略降级，并约束后处理类不抢首轮入口。

## 触发条件

本技能不直接响应用户请求，而是作为路由治理层被 head-teacher 调用。

使用场景：
- 多个 skill 看起来都能处理当前请求
- 新增 skill 后需要防止破坏原有路由
- 出现误触发、错路由、串路由
- 需要专项判断"这句话到底该走哪个 skill"

---

## 路由优先级

| 优先级 | 类型 | Skill | 说明 |
|-------|------|-------|------|
| 1 | 控制类 | mode-switch, evaluation | 优先拦截，不与业务类竞争 |
| 2 | 组合类 | workflow-pack | 次优先，处理组合型请求 |
| 3 | 单技能类 | lesson-planner, presentation, assignment, tutor, admin | 其后，处理单一任务 |
| 4 | 后处理类 | quality-governor, observability | 只后置参与，不抢首轮入口 |

---

## 路由判断流程

```
步骤1：检查是否为控制类请求
IF 匹配 mode-switch 或 evaluation 触发词
   → 路由到对应控制类 skill
   → 结束

步骤2：检查是否为组合类请求
IF 匹配 workflow-pack 触发词 OR 包含多模块组合
   → 路由到 workflow-pack
   → 结束

步骤3：检查是否为单技能请求
IF 匹配某个单技能触发词
   → 检查是否存在 skill 冲突
   IF 存在冲突
      → 按冲突矩阵裁决
   ELSE
      → 路由到对应单技能
   END
   → 结束

步骤4：使用兜底策略
IF 无法明确判断
   → 按关键词兜底规则路由
   → 结束
```

## 控制类路由

### mode-switch

| 属性 | 定义 |
|-----|------|
| **强触发词** | 开启教师助手、关闭教师助手、打开教师助手模式、退出教师助手模式 |
| **路由条件** | 用户明确要求开启或关闭教师助手模式 |

### evaluation

| 属性 | 定义 |
|-----|------|
| **强触发词** | 运行评测、执行回归测试、运行测试用例 |
| **路由条件** | 用户明确要求运行评测或回归测试 |

---

## 组合类路由

### workflow-pack

| 属性 | 定义 |
|-----|------|
| **强触发词** | 备课包、完整备课、一整套课、教学整包、家校沟通包、辅导包、轻量课堂包 |
| **路由条件** | 用户请求包含多个模块组合 |

**组合模式识别**：

| 组合类型 | 触发条件 | 执行顺序 |
|---------|---------|---------|
| 备课包 | "备课包"/"完整备课"/"一整套课" 或 同时包含教案+PPT+练习 | lesson-planner → presentation → assignment |
| 家校沟通包 | "家校沟通包"/"通知+反馈" | admin(通知) → admin(反馈) |
| 辅导包 | "辅导包"/"讲解+练习" | tutor → assignment |
| 轻量课堂包 | "轻量课堂包"/"快速备课" | lesson-planner(简版) → presentation(简版) → assignment(少量) |

## 单技能类路由

### lesson-planner

| 属性 | 定义 |
|-----|------|
| **强触发词** | 教案、备课、课堂设计、教学设计、备一节课 |
| **路由条件** | 用户明确只要教案或备课内容，无组合需求 |

### presentation

| 属性 | 定义 |
|-----|------|
| **强触发词** | PPT、课件、幻灯片、讲课课件、讲课页 |
| **路由条件** | 用户明确只要PPT或课件内容，无组合需求 |

### assignment

| 属性 | 定义 |
|-----|------|
| **强触发词** | 作业、练习、习题、巩固题、分层题、出题、出几道 |
| **路由条件** | 用户明确要求出题、生成练习或作业，无讲解需求 |

### tutor

| 属性 | 定义 |
|-----|------|
| **强触发词** | 讲题、讲解、错因、辅导、分析错误、怎么做、解题思路、不要直接给答案 |
| **路由条件** | 用户要求讲解、辅导或分析错误 |

### admin

| 属性 | 定义 |
|-----|------|
| **强触发词** | 通知、评语、反馈、家校沟通、家长会、班级总结、写给家长、发群 |
| **路由条件** | 用户要求生成单独的通知、评语或反馈，无组合需求 |

## 冲突裁决

### 冲突裁决原则

1. **控制类优先**：控制类 skill 一律优先于业务类 skill
2. **组合优先于单技能**：组合型请求优先路由到 workflow-pack
3. **明确优先于模糊**：用户明确指定时，以用户指定为准
4. **后处理不抢首轮**：后处理类 skill 不得抢占首轮主路由

### 冲突矩阵

#### lesson-planner vs workflow-pack

| 场景 | 路由目标 | 裁决依据 |
|-----|---------|---------|
| 单独备课/教案 | lesson-planner | 无组合需求 |
| 完整备课/一整套课/多模块 | workflow-pack | 组合型请求 |

#### presentation vs workflow-pack

| 场景 | 路由目标 | 裁决依据 |
|-----|---------|---------|
| 纯PPT | presentation | 无组合需求 |
| 组合请求含PPT | workflow-pack | 组合型请求 |

#### assignment vs tutor

| 场景 | 路由目标 | 裁决依据 |
|-----|---------|---------|
| 出题/作业/练习 | assignment | 纯出题需求 |
| 讲解/错因/不要直接给答案 | tutor | 讲解需求 |
| 讲解 + 练习 | workflow-pack | 组合需求（辅导包） |

#### admin vs workflow-pack

| 场景 | 路由目标 | 裁决依据 |
|-----|---------|---------|
| 单独通知/评语/反馈 | admin | 无组合需求 |
| 多模块家校沟通 | workflow-pack | 组合型请求 |

#### 控制类 vs 业务类

- 一律控制类优先

#### 后处理类 vs 主路由类

- 主路由优先，后处理只后置

## 兜底策略

### 关键词兜底

| 关键词 | 兜底路由 |
|-------|---------|
| 备课、教案 | lesson-planner |
| PPT、课件、幻灯片 | presentation |
| 作业、练习、题、习题 | assignment |
| 讲解、错因、不要直接给答案 | tutor |
| 通知、评语、家长、家校沟通 | admin |
| 做一套教学内容 | workflow-pack（通用降级版） |

### 信息不足时的降级

| 缺失情况 | 降级策略 |
|---------|---------|
| 未提供学段 | 默认使用"初中"风格 |
| 未提供年级 | 默认使用"初二" |
| 未提供学科 | 默认使用"语文" |

### 通用工作流降级版

当请求模糊但可能需要组合输出时：

| 工作流类型 | 降级版内容 |
|-----------|-----------|
| 备课包降级版 | 简版教案 + 精简PPT + 少量练习 |
| 家校沟通降级版 | 通用通知 + 简短反馈 |
| 辅导降级版 | 简要讲解 + 3道巩固题 |

---

## 后处理类约束

### 约束原则

后处理类 skill（quality-governor、observability）**不得**抢占首轮主路由。

### 禁止行为

- ❌ 作为首轮路由目标
- ❌ 抢占业务类 skill
- ❌ 在主任务未完成时介入

### 正确流程

```
1. 首轮路由 → 业务类 skill（控制类/组合类/单技能类）
2. 执行主任务
3. 主任务完成后 → 检查是否需要后处理
4. 如需整形 → quality-governor
5. 如需记录 → observability
```

## 路由示例

### 示例1：组合请求

**用户请求**："帮我备课并做个PPT"

**路由判断**：
1. 检查控制类：否
2. 检查组合类：是（备课 + PPT）
3. 路由结果：`workflow-pack`

---

### 示例2：冲突裁决

**用户请求**："帮我讲这道题，再出几道类似的"

**路由判断**：
1. 检查控制类：否
2. 检查组合类：是（讲解 + 出题）
3. 冲突分析：tutor vs assignment
4. 裁决结果：`workflow-pack`（辅导包）

---

### 示例3：关键词兜底

**用户请求**："帮我做个课件"

**路由判断**：
1. 检查控制类：否
2. 检查组合类：否
3. 检查单技能：匹配 presentation（课件）
4. 路由结果：`presentation`

---

### 示例4：控制类优先

**用户请求**："开启教师助手，帮我备一节课"

**路由判断**：
1. 检查控制类：是（开启教师助手）
2. 先执行：`mode-switch`
3. 确认模式开启后，再执行：`lesson-planner`

---

## 边界限制

### 支持

- ✅ 路由判断
- ✅ 冲突裁决
- ✅ 兜底策略
- ✅ 后处理约束

### 不支持

- ❌ 直接响应用户请求
- ❌ 替代 head-teacher 作为主入口
- ❌ 修改路由规则文件
