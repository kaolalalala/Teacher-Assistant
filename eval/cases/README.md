# 测试用例目录

本目录存放固定测试用例，用于回归测试。

## 用例命名规范

- `case-XX-[类型]-[简述].md`
- 例如：`case-01-lesson-plan.md`

## 用例类型覆盖

| 类型 | 说明 |
|------|------|
| lesson-plan | 教案生成 |
| ppt | PPT/课件生成 |
| homework | 作业/练习生成 |
| admin | 行政文案 |
| tutoring | 学生辅导 |
| workflow | 工作流包 |
| mode | 模式开关检查 |

## 用例结构

每个用例应包含：

1. **用例名称**：简明描述
2. **输入指令**：用户会说的完整指令
3. **覆盖场景**：测试的能力点
4. **预期行为**：应该发生什么
5. **检查点**：具体验证项

## 注意事项

- 用例应保持稳定，不要频繁修改
- 用例应覆盖典型场景，不是边界场景
- 用例应独立，不依赖其他用例的执行结果
