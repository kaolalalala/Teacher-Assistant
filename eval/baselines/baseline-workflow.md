# 预期行为基线：工作流包

## 能力名称

workflow-pack（工作流包）

## 核心行为预期

### 任务识别

识别工作流包类型：

| 工作流包 | 触发关键词 |
|---------|-----------|
| 备课包 | 备课包、完整备课、一整套课、教学整包 |
| 家校沟通包 | 家校沟通包、一套家长沟通、通知+反馈 |
| 辅导包 | 辅导包、一套辅导内容、讲解+练习 |

### 执行顺序

| 工作流包 | 执行顺序 |
|---------|---------|
| 备课包 | lesson-planner → presentation → assignment |
| 家校沟通包 | admin(通知) → admin(反馈) → 沟通建议 |
| 辅导包 | tutor → assignment |

## 输出结构要求

### 备课包

必须包含：

1. 教案（完整结构）
2. PPT 课件（逐页内容）
3. 练习（题目+答案）

### 家校沟通包

必须包含：

1. 通知草稿
2. 学习反馈草稿
3. 家长沟通建议（可选）

### 辅导包

必须包含：

1. 错因分析
2. 分步讲解
3. 巩固练习
4. 教师建议

## 禁止事项

- ❌ 执行顺序错误
- ❌ 子产物缺失
- ❌ 各模块风格不统一
- ❌ 存在明显重复或冲突
- ❌ 未做整合直接拼接

## 风格统一要求

工作流包内所有子模块必须：

- 使用相同的学段设置
- 使用相同的教师偏好
- 保持语言风格一致
- 避免重复内容

## 整合输出要求

- 添加导读说明整包内容
- 使用分隔线分隔各模块
- 删除各模块间重复说明
- 统一结尾格式

## 模式相关行为

### 模式 ON 时

- 应读取项目记忆文件
- 所有子模块应体现偏好设置
- 保持偏好一致性

### 模式 OFF 时

- 不默认读取项目记忆文件
- 各子模块按通用规则执行

## 验收标准

### Pass 条件

- 工作流包识别正确
- 执行顺序正确
- 所有子产物完整
- 风格统一
- 整合输出规范

### Partial Pass 条件

- 工作流包识别正确
- 子产物基本完整
- 风格有轻微不统一
- 不影响使用

### Fail 条件

- 工作流包识别错误
- 执行顺序错误
- 子产物缺失严重
- 风格严重不统一
- 未做整合
