# 模型映射表（Model Map）

> 本文件定义每个 Skill 的默认模型档位、可升级档位、可降级档位及不推荐档位。
> 多模型路由稳定化

---

## 一、模型档位定义

| 档位 | 代号 | 适用场景 | 特点 |
|------|------|----------|------|
| 强模型 | `strong` | 复杂编排、组合任务、长输出整合、高一致性要求 | 高质量、高成本、强推理 |
| 标准模型 | `standard` | 大多数稳定文本生成、单技能常规输出 | 平衡质量与成本 |
| 轻量模型 | `light` | 简单改写、简短文案、格式整理、成本敏感场景 | 低成本、快速响应 |

## 三、各 Skill 详细映射规则

### 3.1 workflow-pack（组合工作流）

```
默认档位：strong（强模型）
可升级：无（已为最高档位）
可降级：standard（仅当任务极其简单时）
不推荐：light（轻量模型无法保障整包一致性）

升级触发条件：
- 多 Skill 组合编排
- 长文本整包输出
- 强风格一致性要求

降级触发条件：
- 单一简单任务组合
- 输出长度极短
```

### 3.2 lesson-planner（教案生成）

```
默认档位：standard（标准模型）
可升级：strong（详细教案、长篇教学设计）
可降级：light（简版教案、大纲级输出）
不推荐：无

升级触发条件：
- 完整教案（含目标、过程、评价）
- 长篇教学设计（>2000字）
- 跨学科/跨课时整合设计

降级触发条件：
- 简版教案（<500字）
- 大纲级快速输出
- 仅需框架结构
```

### 3.3 presentation（课件生成）

```
默认档位：standard（标准模型）
可升级：strong（长篇PPT整包、强结构要求）
可降级：light（单页摘要、简单提纲）
不推荐：无

升级触发条件：
- 完整PPT整包（>20页）
- 强结构化要求
- 多媒体内容整合

降级触发条件：
- 单页摘要
- 简单提纲（<10页）
- 仅需框架
```

### 3.4 assignment（作业生成）

```
默认档位：standard（标准模型）
可升级：strong（复杂分层题、多条件题型）
可降级：light（简单基础练习）
不推荐：无

升级触发条件：
- 分层作业设计
- 多条件综合题型
- 创新性/探究性题目

降级触发条件：
- 简单4~6题基础练习
- 标准题型重复
- 仅需题量填充
```

### 3.5 tutor（答疑辅导）

```
默认档位：standard（标准模型）
可升级：strong（复杂启发式讲解、长链条思路拆解）
可降级：无
不推荐：light（轻量模型难以保障辅导质量）

升级触发条件：
- 复杂概念讲解
- 多步骤思路拆解
- 启发式引导设计

降级触发条件：
- 不允许降级（辅导质量优先）
```

### 3.6 admin（行政事务）

```
默认档位：light（轻量模型）
可升级：standard（正式通知、家校沟通套件）
可降级：无
不推荐：strong（行政文本无需强模型）

升级触发条件：
- 正式通知公文
- 家校沟通套件
- 需要正式语气

降级触发条件：
- 不允许降级（已为最低档位）
```

### 3.7 quality-governor（质量治理）

```
默认档位：light（轻量模型）
可升级：standard（复杂整形需求）
可降级：无
不推荐：strong（整形任务不需强模型）

升级触发条件：
- 复杂格式整形
- 多维度质量检查
- 需要语义理解

降级触发条件：
- 不允许降级（已为最低档位）
```

### 3.8 observability（可观测性）

```
默认档位：light（轻量模型）
可升级：无
可降级：无
不推荐：standard, strong（摘要记录无需高档模型）

升级触发条件：
- 不允许升级（成本控制优先）

降级触发条件：
- 不允许降级（已为最低档位）
```

### 3.9 evaluation（效果评估）

```
默认档位：standard（标准模型）
可升级：strong（复杂回归判断、多维度评估）
可降级：light（简单统计汇总）
不推荐：无

升级触发条件：
- 复杂回归分析
- 多维度综合评估
- 需要深度推理

降级触发条件：
- 简单统计汇总
- 单一维度评估
- 仅需数值计算
```

### 3.10 mode-switch（模式切换）

```
默认档位：light（轻量模型）
可升级：无
可降级：无
不推荐：standard, strong（模式口令处理无需高档模型）

升级触发条件：
- 不允许升级（任务简单）

降级触发条件：
- 不允许降级（已为最低档位）
```

### 3.11 router-governor（路由治理）

```
默认档位：standard（标准模型）
可升级：strong（路由冲突复杂、边界模糊严重）
可降级：light（简单路由判断）
不推荐：无

升级触发条件：
- 多 Skill 冲突严重
- 边界判断模糊
- 需要深度推理

降级触发条件：
- 明确单一路由
- 简单关键词匹配
```

### 3.12 model-governor（模型治理）

```
默认档位：standard（标准模型）
可升级：strong（复杂模型选择决策）
可降级：无
不推荐：light（模型判断需稳定性）

升级触发条件：
- 多模型对比决策
- 复杂升级/降级判断
- 成本与质量权衡

降级触发条件：
- 不允许降级（判断稳定性优先）
```

### 3.13 head-teacher（主入口路由器）

```
默认档位：standard（标准模型）
可升级：strong（复杂路由+模型联合决策）
可降级：light（简单明确路由）
不推荐：无

升级触发条件：
- 复杂用户意图理解
- 多 Skill 联合判断
- 模型路由复杂决策

降级触发条件：
- 明确单一意图
- 简单关键词匹配
```

---

## 四、模型选择优先级

1. **优先使用默认档位**：同类任务保持模型一致性
2. **升级需满足触发条件**：避免无意义占用强模型
3. **降级需评估影响**：确保输出质量可接受
4. **禁止使用不推荐档位**：防止质量/成本失控

---

## 五、映射表维护规则

1. 新增 Skill 必须定义默认模型档位
2. 档位调整需记录变更原因
3. 定期审查档位设置的合理性
4. 实验性调整需在 model-policy.md 中记录
