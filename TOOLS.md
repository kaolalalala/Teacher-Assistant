## 输出路径规则

### 教案输出
- **目录**：`outputs/lessons/`
- **命名规则**：`{学科}_{年级}_{课题}_{日期}.md`
- **示例**：`语文_初二_春_20260228.md`

### PPT大纲输出
- **目录**：`outputs/ppts/`
- **命名规则**：`{学科}_{年级}_{课题}_PPT_{日期}.md`
- **示例**：`语文_初二_春_PPT_20260228.md`

### PPT成品内容包输出
- **目录**：`outputs/ppt_files/`
- **命名规则**：`{学科}_{年级}_{课题}_PPT成品_{日期}.md`
- **示例**：`语文_初二_春_PPT成品_20260228.md`

### 行政文案输出
- **目录**：`outputs/admin/`
- **命名规则**：`{文案类型}_{对象}_{日期}.md`
- **示例**：`家长会通知_家长_20260228.md`

### 辅导讲解输出
- **目录**：`outputs/tutoring/`
- **命名规则**：`{学科}_{知识点}_{日期}.md`
- **示例**：`数学_一元二次方程_20260228.md`

### 作业/练习输出
- **目录**：`outputs/assignments/`
- **命名规则**：`{学科}_{年级}_{课题}_练习_{日期}.md`
- **示例**：`语文_初二_春_练习_20260228.md`
- **内容**：结构化题目列表、参考答案、使用建议

## 学段适配规则

### 学段识别

| 学段 | 年级关键词 |
|-----|---------|
| 小学 | 小学、一年级~六年级、1~6年级 |
| 初中 | 初中、初一~初三、七年级~九年级、7~9年级 |
| 高中 | 高中、高一~高三、十年级~十二年级 |

### 默认策略

- 用户明确提供学段 → 使用对应学段风格
- 用户未提供 → 默认使用"初中"风格
- 无法判断 → 生成通用版，提示可补充学段优化

### 学段风格差异

| 维度 | 小学 | 初中 | 高中 |
|-----|------|------|------|
| 整体风格 | 活泼趣味 | 结构清晰 | 紧凑高效 |
| 语言难度 | 简单口语 | 规范准确 | 专业精炼 |
| 内容密度 | 低 | 中 | 高 |
| 互动频率 | 高 | 中 | 适中 |
| 题目难度 | 基础 | 基础+提高 | 综合+能力 |

### 学段适配要点

各模块在生成内容时，必须根据学段调整：

| 模块 | 小学适配 | 初中适配 | 高中适配 |
|-----|---------|---------|---------|
| lesson-planner | 活泼、短环节、多互动 | 结构化、重难点明确 | 紧凑、深度分析 |
| presentation | 字少、图多、节奏快 | 结构清晰、典型例题 | 信息密度高、方法导向 |
| assignment | 题少、简单、生活化 | 典型题、分层设计 | 综合题、能力导向 |
| tutor | 细步骤、多鼓励 | 方法讲解、思路引导 | 深度分析、迁移归纳 |
| admin | 温和亲切、鼓励为主 | 专业诚恳、平衡 | 简洁目标、效率导向 |

## 输出格式

- **默认格式**：Markdown
- **编码**：UTF-8
- **换行**：LF 或 CRLF 均可

## 文件操作规则

1. **不覆盖已有文件**
   - 如果目标文件已存在，添加序号后缀
   - 示例：`语文_初二_春_练习_20260228_2.md`

2. **草稿保存**
   - 如需保存草稿，优先保存到 `outputs/` 对应子目录
   - 草稿文件名添加 `_draft` 后缀

3. **用户明确要求覆盖时**
   - 先备份原文件（添加 `.bak` 后缀）
   - 再写入新内容

## 文件保存流程

### 标准保存流程

```
生成内容 → 检查文件名是否存在 → 保存文件 → 返回保存路径
                ↓
         如存在：添加序号后缀
```

### 教案保存流程

1. **生成文件名**：`{学科}_{年级}_{课题}_{YYYYMMDD}.md`
2. **检查冲突**：如存在，添加序号
3. **保存文件**：路径 `outputs/lessons/{文件名}`
4. **返回信息**：`✅ 教案已保存至：outputs/lessons/语文_初二_春_20260228.md`

### 文件内容规范

每个输出文件应包含：

1. **文件头注释**
   ```markdown
   ---
   生成时间：{YYYY-MM-DD HH:mm:ss}
   任务类型：{教案/PPT成品内容包/行政文案/辅导讲解/作业练习}
   ---
   ```

2. **正文内容**
   - 按模板结构组织
   - 格式规范统一

3. **文件尾注释**（可选）
   ```markdown
   ---
   *本内容由教师助手生成*
   ```

- **路径**：`profiles/teacher-profile.md`
- **用途**：记录教师的长期偏好设置

### 班级画像档案

- **路径**：`profiles/classroom-profile.md`
- **用途**：记录班级整体层面的教学信息
- **特点**：
  - 仅记录整体性、非敏感信息
  - 包含班级基础水平、课堂风格、常见薄弱点等
  - 不记录学生个人敏感信息

### 教师偏好模板

- **路径**：`templates/teacher-preference-template.md`
- **用途**：定义教师偏好档案的标准结构和可选值

### 班级画像模板

- **路径**：`templates/classroom-profile-template.md`
- **用途**：定义班级画像的标准结构和应用规则

### 偏好应用优先级

```
当前用户明确要求 > 教师偏好 > 班级层面画像 > 通用默认规则
```

### 偏好应用规则

1. 偏好必须影响结果，但不能压过当前指令
2. 班级画像只能做"整体调节"
3. 偏好缺失时必须优雅降级，不直接失败
4. 各模块在生成内容前，应优先参考可用的偏好信息

### 各模块偏好应用

| 模块 | 可读取偏好 | 可读取画像 |
|-----|-----------|-----------|
| lesson-planner | 教案风格、语言风格、教学偏好 | 整体基础、教学节奏 |
| presentation | PPT风格、文字密度、互动偏好 | 课堂参与度、互动频率 |
| assignment | 题量、作业风格、分层偏好 | 整体基础、作业完成倾向 |
| tutor | 讲解风格、深度、鼓励偏好 | 整体基础、常见薄弱点 |
| admin | 沟通语气、文案长度 | 家长关注度、沟通偏好 |

## 工作流包配置

### 工作流包模板

- **路径**：`templates/workflow-pack-template.md`
- **用途**：定义工作流包的标准结构和执行规则
- **特点**：
  - 定义备课包、家校沟通包、辅导包、轻量课堂包
  - 包含触发词、执行顺序、降级策略

### 工作流包技能

- **路径**：`skills/workflow-pack/SKILL.md`
- **用途**：工作流包的识别、编排和执行逻辑
- **特点**：
  - 不替代基础技能，而是组合调度
  - 支持裁剪和降级

### 工作流包类型

| 工作流包 | 包含模块 | 执行顺序 |
|---------|---------|---------|
| 备课包 | 教案 + PPT + 练习 | lesson-planner → presentation → assignment |
| 家校沟通包 | 通知 + 反馈 + 建议 | admin → admin → 沟通建议 |
| 辅导包 | 讲解 + 练习 + 建议 | tutor → assignment |
| 轻量课堂包 | 简版教案 + 简版PPT + 少量练习 | lesson-planner → presentation → assignment |

### 工作流包输出规则

1. **组合调度**：工作流包由已有模块产物组合而成，不新增独立输出目录
2. **落盘路径**：各子模块仍使用原有输出目录
   - 教案 → `outputs/lessons/`
   - PPT → `outputs/ppt_files/`
   - 练习 → `outputs/assignments/`
   - 行政 → `outputs/admin/`
   - 辅导 → `outputs/tutoring/`
3. **整合展示**：主 agent 负责最终整合展示顺序

### 工作流包执行规则

1. **串行执行**：所有子模块必须串行执行
2. **上下文传递**：前一模块输出摘要传递给后续模块
3. **风格统一**：所有子产物使用相同学段风格和语言风格
4. **可裁剪**：用户明确裁剪需求时，调整工作流内容
5. **优雅降级**：信息不足时使用降级策略

### 评测结果分类

| 结果 | 说明 |
|------|------|
| Pass | 核心行为符合预期，输出结构完整，无明显退化 |
| Partial Pass | 能完成任务，但结构/风格/顺序有轻微偏差，不影响主能力可用性 |
| Fail | 路由错误、结构缺失严重、模式失效、工作流顺序错误、核心功能不可用 |

### 评测原则

1. **不做逐字比对**：评测标准不是"和参考文本完全一致"
2. **重点检查行为与结构**：路由、结构、顺序、模式是否正确
3. **允许轻微措辞变化**：只要行为正确、结构合理，即可视为通过或部分通过
4. **评测结果必须可追溯**：每轮评测都留下结果文件

| 教案生成 | Tier-1 | 核心产出 |

| 输出整形 | Tier-2 | 格式化任务 |

### 模型路由技能

- **路径**：`skills/model-router/SKILL.md`
- **用途**：模型选择、降级管理、路由决策

### 降级策略

当目标模型不可用时：

1. 尝试备选模型
2. 若备选也不可用，使用当前可用主模型
3. 记录降级事件到 `logs/errors/`
4. 不因模型路由失败而中断主流程

### 路由优先级

```
1. 用户明确指定模型
2. 任务类型默认映射
3. 系统负载状态调整
4. 备选模型降级
5. 兜底：使用当前可用主模型
```

### 模型路由原则

1. 路由是"按任务类型"而不是"随机选择"
2. 默认映射必须可读、可修改
3. 模型路由不应改变业务规则本身
4. 后续评测应能观察模型变更是否带来回归退化
5. 即使当前只有 1~2 个模型，也要保留逻辑分层，方便后续替换

- **用途**：路由判断、冲突裁决、兜底策略执行

### 路由治理原则

1. 所有请求先经过总路由判断，不允许由各 skill 自由竞争
2. 固定路由优先级：控制类 → 组合类 → 单技能类 → 后处理类
3. 多个 skill 同时可能命中时，必须按冲突矩阵裁决
4. 信息不足时，必须按统一兜底策略处理
5. 后处理类不能抢占首轮主路由

### 模型规则目录

| 目录 | 用途 | 内容特点 |

| `models/` | 模型规则目录 | 高优先级模型路由治理规则 |

### 模型档位定义

| 档位 | 代号 | 适用场景 | 特点 |
|------|------|----------|------|
| 强模型 | `strong` | 复杂编排、组合任务、长输出整合 | 高质量、高成本、强推理 |
| 标准模型 | `standard` | 大多数稳定文本生成、单技能常规输出 | 平衡质量与成本 |
| 轻量模型 | `light` | 简单改写、简短文案、格式整理 | 低成本、快速响应 |

### 模型路由原则

1. 所有请求在完成 skill 路由后，应进入模型路由判断
2. 模型路由必须优先遵循"按 skill 绑定默认模型"的原则
3. 不允许同类任务无规则频繁换模
4. 模型选择必须兼顾：任务复杂度、输出稳定性、成本控制、风格一致性
5. 当默认模型不适合时，必须按升级/降级规则调整

### 模型升级规则

| 触发场景 | 判断标准 | 升级目标 |
|----------|----------|----------|
| 组合型工作流 | 多 Skill 编排、整包输出 | strong |
| 长文本整合 | 输出预计 >2000字 | strong |
| 结构复杂 | 多层级、多模块结构 | strong |
| 强一致性要求 | 风格/格式需高度统一 | strong |
| 复杂推理 | 多步骤逻辑链 | strong |

### 模型降级规则

| 触发场景 | 判断标准 | 降级目标 |
|----------|----------|----------|
| 任务简单 | 复杂度 < 中 | light |
| 输出短 | 预计输出 <500字 | light |
| 格式整理 | 仅需结构调整 | light |
| 简单改写 | 文本润色、格式化 | light |
| 状态切换 | 模式口令处理 | light |

### 模型回退策略

当模型不可用时，按以下链路回退：

```
strong → standard → light → 错误
```

回退时必须：
- 保持输出结构不变
- 告知用户当前使用的模型档位
- 记录回退事件到日志
